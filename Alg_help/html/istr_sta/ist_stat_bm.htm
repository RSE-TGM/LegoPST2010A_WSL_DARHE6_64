Backtrack management

This is the interface through which the instructor can go back to previous sessions, access data recorded in the backtrack snapshots automatically recorded, load the backtrack snapshots themselves for restart the simulation or to "replay" the recorded sessions.

It is activated by clicking the Recording -> Backtrack Management button, or, more quickly, through the selection of the B button shortcut in the master menu interface.

 Interface description

 

 

The interface is divided into 4 zones:

 

the menubar

the replay area

the backtrack snapshots scrolling list

the command area, at the bottom side of the window

 

The command area contains the following buttons:

 

LOAD to load the selected backtrack in the simulator database

INIT to validate the last loaded backtrack as the one that will be the starting point of a new training session

VIEW to initiate the conversation that allows to select and view the recorded values of patterns of simulator variables

QUIT to terminate the interface

 

The same commands are available from a popup menu activated by clicking the rightmost mouse button while the pointer is in the backtrack list area.

 

The menubar provides the following commands:

 

File ->

Quit

to terminate the interface

Edit ->

Timer Recording

to modify the time between the automatic snapshots

 

 

 

 

The replay area shows which backtrack was last loaded, either manually, or automatically by the replay function, and a button that permits to activate and deactivate the simulator replay function.

 

 

Interface operation

The interface is used to view and load backtrack snapshots, for purposes of initialize new sessions or to replay recorded sessions.

The interface is not available when no records are recorded in the backtrack file.

Loading backtrack records

The LOAD buttons are available when:

 

the simulator is in the FREEZE, RUN or BACKTRACK state and a backtrack record is selected on the list.

 

To load a backtrack snapshot into the simulator database:

 

select a backtrack record from the list

click on one of the provided LOAD buttons

 

The request activates the following sequence:

 

the simulator state is switched to FREEZE

the current simulator database is saved on I.C. n. 60 (default snapshot)

the simulator state is switched to BACKTRACK

the selected backtrack snapshot is actually loaded

the backtrack interface is updated

 

 

Initializing from a backtrack snapshot

The INIT buttons are available when:

 

a backtrack snapshot has been successfully loaded into the simulator database and the simulator is in BACKTRACK state

 

To initialize the simulator from a backtrack snapshot:

 

click on one of the provided INIT buttons

 

This causes the interface to close and the simulator state to switch to the FREEZE state.

 

 

View the variables value recorded into a backtrack snapshot

The VIEW buttons are available when:

 

a backtrack snapshot is currently selected from the list

 

To inspect the values of the recorded variables:

 

select a backtrack snapshot

click on one of the provided VIEW buttons

 

At this time a new window will appear that shows the values of up to 10 variables. From this interface the instructor can configure and select different patterns of variables using a filter function. See the appropriated paragraph to know more about this interface.

 

 

Restoring the previous session

The instructor has the possibility to restore the session left before entering BACKTRACK mode.

The function is activated by the selection of one of the QUIT buttons after at least one backtrack snapshot has been loaded (the original database has been replaced).

The protocol presents a warning that advise the instructor. Choose the OK pushbutton to continue or the Cancel pushbutton to abort the operation.

If confirmed, the function replaces the current simulator database with the I.C. n. 60 (the one recorded when first entered the BACKTRACK state), and the simulator state is switched to FREEZE.

 

 

Changing the recording frequency

It is given the possibility to modify the interval at which the instructor station triggers the recording of the backtrack snapshots.

The function is activated by clicking the Edit -> Timer Recording button: an interface is opened with the current value of the interval.

The instructor can modify that value and click the OK button to activate the new value.

 

 

Replay mode

The replay button is available whenever a backtrack snapshot has been loaded into the simulator database.

 

To activate the replay mode:

 

load a backtrack snapshot

click on the Replay Mode button

 

The simulator state switches from BACKTRACK to REPLAY state. In this state the models are running, the frame around the status label on the master menu interface is blinking, and all the commands that the models received during the previously recorded session are repeated to the simulator at the appropriated time.

 

While in this state the automatic recording functions are inhibited and the users should avoid the use of the commands.

 

The replay mode terminates automatically at the end of a session, or when the instructor toggles the Replay Mode button; at this point the simulator state switch back to BACKTRACK.

 

 

Exit the dialog

There are three ways to terminate the interface:

 

using the INIT buttons, to initiate a new session starting from a backtrack snapshot

using the QUIT buttons after a backtrack snapshot has been loaded, to restore the previous session

using the QUIT buttons before loading of a backtrack snapshot to simply quit the interface
