MMI activation

To activate the mmi interface on the instructor station machine:

 

log in whit the same account used to startup the simulator

change directory to the location where the mmi pages are located (i.e. $HOME/sked/cc_puerto/globpages)

type mmi &

 

 

Alternatively, a command such the following can be configured with the options interface:

 

cd $HOME/sked/cc_puerto/globpages ; mmi &

 

This command will have the same effect of the previous procedure.

 

 

To activate the mmi interface on a machine other the instructor station:

 

log in the machine with the correct account

change directory to the location where the mmi pages are located (i.e. $HOME/sked/cc_puerto/globpages)

type mmi &

 

The Host field of the Context.ctx file located in the mmi directory should contain the name of the machine running the simulation.

 

 

Manual shutdown

Should the normal procedure fail to shutdown the simulator (due to UNIX problems), follow this manual list:

 

log in whit the same account used to startup the simulator

execute the command killsim

check if there are simulation process still active; if any, kill them using the kill UNIX function
