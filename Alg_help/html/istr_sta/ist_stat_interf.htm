Master menu interface

The master menu is always present at the instructor station interface and allows to initiate all the functions provided by the instructor station.

Interface description

 

 

The interface is divided in the following parts:

 

a menubar, through which it is possible to initiate all the instructor station functions

 

a two button command area, that give the possibility to run, freeze and step the simulator

 

a display area, that shows the simulator current state, the simulation time, the delay time and the speed of the simulator

 

a command area, that provides direct access to some of the instructor station functions

 

a message list area, where are listed all the commands issued by the instructor

 

the title bar that specifies the session title (as written by the instructor) and where the database was coming from (I.C., B.T. or steady state)

When the simulator is in RUN or REPLAY status, a frame situated around the status label will blink at the simulation executing frequency.

 

 

Interface operation

The master menu interface becomes available once the instructor has request the startup of the simulator. When popped up, the interface will show that the simulator is in the STOP state (because the models are non initialized yet), and the buttons are not available.

At the end of the initialization procedure the simulator goes in FREEZE status, the simulation time field shows the current time and the buttons become sensitive.

At that time the instructor can RUN the simulation starting from the steady state or from an I.C. or a B.T. (if any); if sessions were previously recorded, the REPLAY function is also available.

 

The selection of the buttons presented by the menubar will drive the instructor station to activate the available interfaces.

 

 

Exit the interface

The master menu interface is terminated by the Instructor Station interface when either the SHUTDOWN or Quit function is requested.

Interface description

 

 

The window shows five fields for each of the 60 possible Initial Conditions:

 

the I.C. number

the padlock, displayed if the I.C. is protected

the simulation time at which the I.C. was snapped, or blank if the I.C. is available

the comment the instructor entered when the I.C. was snapped, or the phrase <<< AVAILABLE >>> if the I.C. is not snapped

the "previous topology" icon, if the snapshot was recorded before the last time the simulator has been compiled.

 

The bottom side of the window contains the following buttons:

 

LOAD to load the selected I.C. in the simulator database, making it the starting point of a new training session

SAVE to snap the present simulator database into the selected I.C.

VIEW to initiate the conversation that allows to select and view the recorded values of patterns of simulator variables

QUIT to terminate the Initial Condition interface

 

The same commands are available from a popup menu activated by clicking the rightmost mouse button while the pointer is in the snapshot list area.

Saving Initial Conditions

The SAVE buttons are not available when:

 

the simulator state is STOP or ERROR

the I.C. selected as destination of the snapshot is protected by the padlock and the instructor is not SUPERUSER

 

and are available in all the other cases.

 

To save the simulator database into an I.C.:

 

select a valid I.C. from the list

click on one of the provided SAVE button

answer to the warning message if the I.C. is busy

enter the comment and push the SAVE button

 

The save initial condition protocol presents a warning message if a snapshot is already present in the destination I.C.. Choose the OK pushbutton to continue or the Cancel pushbutton to abort the operation.

Before the I.C. is actually saved the instructor is prompted to enter his own comment, that will be the phrase displayed in the comment field of the I.C. list. Choosing the SAVE pushbutton the operation will take place, while the QUIT pushbutton is the last chance the instructor has to abort the operation.

After the operation has been successfully executed the I.C. list is refreshed.

 

 

Loading Initial Conditions

The LOAD buttons are available when:

 

the simulator is in the FREEZE state and a valid snapshot is recorded in the selected I.C. (snapshot recorded and not flagged as "previous topology")

 

and are not available in all the other cases.

 

To load an I.C. into the simulator database:

 

select a valid I.C. from the list

click on one of the provided LOAD buttons

 

The function is immediately executed and the new content of the database can be explored by looking at the mimics interface.

The initial condition interface remain active for further operations.

 

 

Remove Initial Conditions

The CUT button is available when:

 

the selected I.C. contains a valid snapshot and it is not protected by the padlock

the selected I.C. contains a valid snapshot and the instructor has SUPERUSER privileges

 

and is insensitive in all the other cases.

 

To remove an I.C. from the list:

 

select a valid I.C. from the list

click on the Cut button

answer to the warning message

 

The cut initial condition protocol presents a warning message. Choose the OK pushbutton to continue or the Cancel pushbutton to abort the operation.

After the operation has been successfully executed the I.C. list is refreshed.

The removed I.C. has been copied into an Instructor Station buffer, and is available for subsequent Paste operations.

 

 

Copy an Initial Conditions into the local buffer

The COPY button is available when:

 

the selected I.C. contains a valid snapshot (snapshot recorded and not flagged as "previous topology") and is insensitive in all the other cases.

 

To copy an I.C. into the local buffer:

 

select a valid I.C. from the list

click on the Copy button

 

The function is immediately executed: the I.C. is copied into an Instructor Station buffer, and is available for subsequent Paste operations.

 

 

Paste the local buffer into an Initial Condition

The PASTE button is available when:

 

the local buffer has been filled from a previously executed Cut or Copy operation and the selected I.C. is not protected by the padlock

the local buffer has been filled from a previously executed Cut or Copy operation and the instructor has the SUPERUSER privileges

 

and is insensitive in all the other cases.

 

To paste the local buffer into an I.C.:

 

fill the local buffer by copying or removing an I.C.

select the destination I.C.

click the Paste button

answer to the warning message if the I.C. is busy

 

The paste I.C. protocol presents a warning message if a snapshot is already present in the destination I.C.. Choose the OK pushbutton to continue or the Cancel pushbutton to abort the operation.

After the operation has been successfully executed the I.C. list is refreshed.

 

 

Validation of "old topology" Initial Conditions

The check button is available when:

 

the selected I.C. is flagged as "previous topology" and the instructor has the SUPERUSER privileges

 

and is insensitive in all the other cases.

 

To accept as valid a "previous topology" I.C.:

 

select a flagged I.C.

click on the Check button

answer to the warning message

 

The Check I.C. protocol presents a warning message. Choose the OK pushbutton to continue or the Cancel pushbutton to abort the operation. After the operation has been successfully executed the I.C. list is refreshed. The "previous topology" flag is not present anymore and the I.C. is available for normal operations.

 

 

View the variables value recorded into an Initial Condition

The VIEW buttons are available when:

 

a valid snapshot is recorded in the selected I.C. (snapshot recorded and not flagged as "previous topology")

 

and are not available in all the other cases.

 

To inspect the values of the recorded variables:

 

select a valid I.C.

click on one of the provided VIEW buttons

 

At this time a new window will appear that shows the values of up to 10 variables. From this interface the instructor can configure and select different patterns of variables using a filter function. See the appropriated paragraph to know more about this interface.

 

 

Exit the dialog

To close the interface select one of the QUIT buttons provided by the interface.

 

 

View interface

This function allows the instructor to read the value of all the simulator variables as they are recorded into a snapshot or backtrack record.

 

The interface is activated selecting one of the VIEW buttons provided in the snapshot and backtrack management interfaces.

The instructor can have any number of VIEW INTERFACES opened at any given time.

Deleting items from the list

To remove one variable at a time from the displayed list:

 

select an entry clicking the toggle button at the left side of the window

click on the Cut button

 

The list will be immediately updated.

 

To clean the entire list:

 

click on the Clear all button

 

The list will be immediately updated.

 

 

Selecting new variables

To select new variables to fill the displayed list:

 

activate the filter selecting the Select new ... button

select a position in the list clicking the toggle button at the left side of the window

choose a variable in the filter interface

push the SELECT button in the filter interface

 

The list will be immediately updated.

 

The selection is rejected if no position in the list is selected.

No protection exists against the replacement of occupied position in the list.

See the appropriated paragraph to know more about this interface.

 

 

Saving the configuration files

The present set of variables can be stored in a file for future use; to make it the default set (i.e. the one first accessed when calling the interface):

 

click on the Save default button

 

To save the set present on display:

 

click on the Save as ... button to activate the file selection interface

enter a new file name or select an existent one if it is desired to overwrite that file

select OK on the file selection interface

 

It is recommended to respect the termination suggested by the interface.

 

 

Opening the configuration files

The list of the variables the values of which are to be displayed is stored in a default file by the instructor station.

However, the instructor has the possibility to save different sets of variables in different files.

 

To load the default configuration file:

 

push the Open default button

 

The list will be immediately updated.

 

To load an instructor configured pattern of variables:

 

push the Open ... button to activate the file selection interface

choose a valid file from the list

select OK in the file selection interface

 

The list will be immediately updated.

The interface is divided into 4 main zones:

 

the menubar

the filter area

The commands provided by the menubar are:

 

File

to provide filtering by models & blocks

to provide filtering by model

Filter

to provide filtering by KKS tag

 

Filter

to provide filtering by type of variable (temperature, pressure, etc)

 

 

 

 

 

The filter area presents the filters currently selected.

 

The variable list area lists all the variables that currently satisfy the filter selection criterion. Furthermore, a "find" function is available for string search.

 

The command area presents two buttons:

 

SELECT to transfer the selected variable to the VIEW list

QUIT to terminate this dialog

 

 

Interface operation

Variable selection

The interface is activated with the KKS filter activated by default; in that way all the variables of the simulator that have been tagged with a KKS label are displayed. Further selection among the displayed variables is possible by typing characters in the KKS FILTERS fields and clicking on the APPLY button.

 

Using the menubar, the instructor can activate the Models & Blocks filter or the Models Only filter (they are both alternatives to the All Variables default filter).

In the first case the lists of the models constituting the simulator is displayed along with, for each model, the list of the respective blocks. At the bottom of the Blocks list a "find" function is provided for string search.

In the latter case it is displayed the list of the models only.

 

The activation of the By Type filter makes available the part of the interface that allows to restrict the visibility to the variables that match the type selection (no type selected means no filter applied); further selection is available filtering by Input, Output and Not connected input variables. The APPLY button starts the procedure that updates the variables list area.

 

The selection is made at the variables list area, eventually using the provided "find" function, clicking on the desired item of the list.

 

 

The SELECT button transfers the selected variable to the tagged entry of the VIEW interface.

 

Exit the dialog

To close the interface select the QUIT button provided by the interface.
