Superuser privileges

The startup procedure allows the instructor to activate the simulator as normal user or as superuser.

In the latter case the instructor has the possibility to protect Initial Conditions, to override these protections and to validate Initial Conditions flagged as invalid after a simulator compilation.

Simulator start-up and shutdown

In the simulator configuration, based on three workstations, one will be used as master and the others as slaves.

The simulator is activated at the master machine with a single command; all the commands and functions needed to initialize, configure and use the simulator are available from the graphical interface of the Instructor Station console.

The instructor is required to type a command at the keyboard of the Instructor station machine. All the software will be initialized and the simulator will come up in the FREEZE state.

 

The initialization procedure restores the simulator situation left before the last shutdown: the simulation time is recovered from the recorded files and the last backtrack record can be used to load the last valid database.

 

Change of the simulator state

All the permitted transitions between the states of the simulator are manually driven by the instructor.

 

While the RUN state is selected, the instructor is prompted to enter a "title" of the session that will start from this moment on, if the selection comes after a database replacement (initial condition or backtrack snapshot loading).

 

 

Snapshot management

The instructor has the possibility to record, load and maintain up to 60 initial conditions.

 

An appropriate interface is provided that make possible to:

save a snapshot of the simulator database as an initial condition chosen by the instructor along with a comment. This functionality is available from all the simulator states.

load the simulator database from a selected initial condition. This functionality is available from the FREEZE state only.

delete, move and copy initial conditions, as well as modify the comments associated to the initial conditions. This functionality is available from all the simulator states.

view the values of all the simulator variables as they are recorded into the initial conditions records, on lists provided and maintained through the instructor station interface. This functionality is available from all the simulator states.

 

Backtrack management

An automatic function is provided that, at the time interval specified by the instructor, records the simulator database into a file in circular fashion. The file is sized for 120 records.

An interface is provided that make possible to:

view the values of all the simulator variables as they are recorded into the backtrack records, on lists provided and maintained through the instructor station interface. This functionality is available from all the simulator states.

manually select a backtrack record for loading into the simulator database. The first time such a load is requested the simulator is automatically switched to the backtrack state.

switch from the BACKTRACK state to the REPLAY state, and back to BACKTRACK state. The REPLAY state stops automatically at the session limit.

switch from the BACKTRACK state to the FREEZE state in two possible ways:

by validating the database substitution made while in BACKTRACK state

recovering the situation left while entering in BACKTRACK state

The backtrack snapshot list is displayed by record time order, and are clearly pointed out the start and the end of the sessions.

The file can be cleared by using the appropriate function.

 

 

Record and replay

All the events coming from outside the models that affect the simulation (generally: modification of input variables of the models, such as trainee actions) are automatically recorded. In REPLAY mode these "perturbations" are submitted to the models at the appropriate time, so producing an effective "replay" of a previous session. The REPLAY mode stops automatically at the session limit or on instructor’s request.

 

 

Time scaling

An interface is provided that make possible to modify the "speed" of the simulator by introducing a multiplier of the system clock. Requesting the "maximum speed", the scheduler is forced to run the models at the maximum frequency that don’t causes the delay time to increase.

This feature is available from all the simulator states.

 

 

Continuos recording

An interface is provided that make possible to select from the simulator database up to 1000 variables that will be recorded, at a fixed frequency, on an apposite circular file.

A trend function is available to display the recorded values; an interface permits the configuration of up to 100 trends. The instructor has the possibility to modify on line the preassigned parameters that define the trends (time scale, engineering units and scale of the displayed variables and the variables themselves). All these functions are available from all the simulator states.
