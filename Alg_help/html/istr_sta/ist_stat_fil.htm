User environment

The instructor logs in the instructor station machine with the account used to install the simulator.

The environment variables provide paths for executable programs and for location where instructor station software stores special files.

The .Xdefaults file located in the user login directory provides resource settings for the instructor station graphic interface.

If the installation process ran without errors and no modifications are requested from the standard installation the user does not need any configuration of these 2 files.

 

 

 

 

 

Instructor Station files

During the training session the instructor station software uses some files for log and configuration purposes.

They are:

 

.message_list

contains the log of all the operations the instructor made since the simulator startup; it is the copy of the master menu message list.

 

.preview

contains the list of the 10 default variables used by the view function.

 

*.view

files created by the instructor to hold sets of variables used by the view function.

 

.bi_options

holds the informations related to the user configurable portion of the instructor station interface.

 

*.c_rec

files created by the instructor to hold sets of variables used by the continuos recording function as alternative to the default file (recorder.edf).

 

 

 

 

 

 

 

 

All the above files are stored in the directory specified by the environment variable $BANCO_FILES. They are created and maintained by the instructor station interface.

 

 

Other files used by the instructor station software, that reside in the directory where the simulator is initialized, are:

 

S01

contains the simulator topology

 

S02_OSF1

contains the simulator topology database

 

variabili.edf

variabili.rtf

contain the ASCII and binary variables database

 

 

recorder.edf

recorder.rtf

contain the ASCII and binary database for the continuos recording function

 

snapshot.dat

holds up to 60 initial conditions. If not existent, it is created at startup time.

 

backtrack.dat

holds up to 120 time based automatically recorded snapshots. If not existent, it is created at startup time.

 

perturbazioni.dat

records all the commands to the models issued by the instructor and the trainee. If not existent, it is created at startup time.

 

f22circ.hd f22circ.dat

f22_fgraf.edf

set of file that hold the header and the data recorded by the continuos recording function. The last one records the groups of variables as they are aggregated by the user during the view function. If not existent, they are created at startup time.
