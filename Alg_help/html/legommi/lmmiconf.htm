LEGOMMI: Configuration

The configuration phase consists in the creation and construction of a page. The user can create and modify a page calling the tool named "config" both for synoptic and regulation scheme. A number of Editors are available.

 

Creation

The user can create a new page selecting "Page" or "Regulation" on the Create pulldown menu. MMI will then display a window where basic information on the page geometry can be inserted. With the first choice MMI creates a synoptic and with the second a regulation page.

For regulation schemes management every page is associated with a progressive number, formed by numbers and characters, i.e.:

00,01, ... , 0A, 0B, ... , 0Z,10,11, ... , ZZ

These numbers are not re-computed after deleting a page.

Construction

First, the user must instantiate one or more objects retrieved from a library and then select a graphic object to assign the resources using the attributes configurator named Resource Editor.

Furthermore the user can complete the page with the background by using the Draw Editor.

In construction of regulation schemes once instantiation has been completed the user has to connect the objects's ports. Snap is available to facility the building of connections .

Configuration: Resource Editor

The Resource Editor allows to view and to edit the resources of every object both in the page and in the library.

 

Not editable resources are also displayed but specified buttons and text fields are insensitive, as some resources are editable in libraries but not on page.

The Resource Editor is called by the user once the object has been selected!PopupId(`gloshelp.hlp',`lgc_gl_gph_select') by choosing Resource on Edit's pulldown menu or Resource on the pop-up menu (which is recalled from the third mouse button pressed on page/library).

LEGOMMI displays a window containing the page name and class of the parent, a pulldown menu, the name of the object being edited, an option menu to choose the resource class, the list of the selected resources and the "Apply" and "Load" (Resources) buttons.

"Geometric" are standard resources that define all the geometrical characteristics of the object (X, Y, Width, High, Rotate, Transparent) and some colors.

"Specific" are typical of every kind of object and contain resources like the object type (i.e. pointer, bar for an indicator etc.).

"Color attributes" define the object's typical colors.

The other four ones, if available for the object, allow to associate an object portion with different colors depending on the values taken on by the variable chosen.

Every group is set by selecting an option on the option menu and displayed inside a scrolled window. The above figure displays the "Specific" resources group of the icon.

Every row contains a resource but there are different displaying modalities.

The button and text field are available when an Editor is necessary (Color, Font, Bitmap) to insert the modification. The button, featuring a label with the resources name, activates the editor while the text fields display the resource.

The label features the name and the text field when the user can directly insert the resource value.

The label features the name and the option menu to ascribe the resource value (i.e. a type or a rotation).

Once a resource has been modified to the right of the text field a little line is displayed.

Through the "Apply" push button the Resource Editor sets the current resource directly on the object. Through "Load" (Resources) push button the resources of the currently-selected object are loaded on the same window. The same operation is available also through the File's pulldown menu with "Load Resources". The other options, -"Reset resources" and "Close"-, restore the condition prior to the editing of the individual resources and close the page.

Configuration: Variable editor

Through the Variable Editor the user can select a variable from the set data base (simulator label on the config main window). The Editor reaches the variable by selecting first the module and the module block and then the block variable.

Variable Editor main window:

The three selections above can be made by pressing the "List" push buttons. Once the List push buttons have been clicked, selection windows will be displayed. They contain respectively the list of models, the list of blocks of the selected model and the variable of the selected block.

First the user must choose the Model and the block then, by clicking the second "List" push button. The window which will be displayed features two lists containing the modules (and also the "All" choice) and the blocks respectively. If the user selects a particular module and then clicks "Filter" the block list will be updated only with the blocks of that module; if "All" has been selected, all the model blocks will be displayed.

In the regulation model construction regulation scheme is like a module and coincides with a model block.

After the model and the block have been selected the user can choose the variable.

In order to easily configure a desired variable a number of filters are available.

The filters "Component" and "Sub System" can be activate through an option menu and "All" will deselect filters. These filters, if activated, reduce the number of elements displayed inside model and block lists, according to the Component or Sub System chosen.

Variable Editor allows to load configuration information on a variable related to an object of the drawing area which has already been selected by pressing the "Grab Selected Widget" push button.

When the editor is called during the configuration of an interactive object such as a button it is possible to assign a perturbation. Available perturbations are Not, Step, Pulse, Ramp, UpDown.

Configuration: Color editor

The Color Editor is called every time colors are assigned and displays a window named "Resource Color Viewer" which contains 12 (max) configured colors.

{bmc mi_cores.bmp}

Every color can be selected by clicking on the button which contains the desired color; the adjacent text field contains the string defining the color.

Through the Edit pulldown menu the user can call the "Color Editor" to build a new color, to display the active Colormap or by selecting "Grab color" to import a new one (for example from the color map).

It is possible to change a default color from the default palette by following these steps:

1) select the desired color on the default palette;

2) modify the color either through the Color Editor or by writing the color string in the text field or grabbing a color;

3) press the "Apply" push button;

4) select a second time the button color of the palette;

The new default color palette will be saved in a local file named FileColoriDefault after pressing "OK". This file will be loaded every time the Color viewer is used.

The definition of a new color by the Color Editor is made by dragging the mouse on the red, green and blue scrolling bars. Once the user has pressed "OK" the new color will be exported in the Color Viewer, the display will be possible after pressing "Apply" in the "Color Viewer".

After pressing a color inside an application windows the user can select "Grab color" to capture the color of the point and load it in the Color Viewer.

The Color Map displays the available color of the active colormap. Every color can be captured by using "Grab Color".

Configuration: Font editor

When the user has to assign a text inside a resource, he can change the font by calling the Font Editor.

LEGOMMI displays a window named "Font Viewer" featuring a number of default font which can be selected by the user clicking a push button, a text field which contains the font define string and three confirm-action push button.

The user can edit a new font by using the "Edit Font" on the Edit pulldown menu.

It is possible to change a default font by following these steps:

1) select the default font which has to changed;

2) modify the Font through the Font Editor or writing the desire font string;

3) push the Apply push button of the Font Editor;

4) select a second time the push button of the changed font.

By clicking the OK push button of the Font Viewer the default font will be saved in the FileFontDefault file which is red at every recall of this editor.

The Font Editor shows all the Font available.

The font will be results of 3 choices between these groups:

"Family or Short name",

"Font",

"Size and Charset"

During font selection, an example of the selected font is displayed and once "OK" push button is clicked, Font Editor exports to the Font Viewer the new font which can be shown only after the user has pressed the Font Viewer "Apply" push button.

Configuration: Draw editor

The Draw editor is used in two different cases: during page for background editing or during icon construction to build animated icon drawing.

Functionality provided are the same except for case of enumerated animated colors that are allowed only for animated icon draw editing.

A title on the Draw main window is displayed: "Draw on Page XXXX" or "Draw on animated icon". This editor is called from the "Background Draw" pulldown menu of the page (first modality) or during an icon configuration, in particular by assigning the "Filename" resource which will display a page containing the list of the available (and amendable) icon files; by choosing Edit (or Create for a new file) on the File pulldown menu of this window the editor working with the selected file is called.

In general through this editor the user can draw lines, circles, squares polygons, and every other free line with the mouse or he can modify already objects drawn (dimension, colour, filling).

The kind of object which will be instantiate can be chosen through a series of push buttons, the same applies to fill and line color. Four push buttons (displaying four grey color) are dedicated to set color change variables during icon draw configuration.

Utilities such as Zoom, Snap, Group, Ungroup, are also, provided.

Configuration: zoom and snap

SNAP

By pressing "Set Snap" on the Option pulldown menu of the page, the user can fix a grid where to draw the line and obtain linear connections.

This option allows to build a linear connection between ports.

The user can set the snap value in pixels.

ZOOM

In order to facilitate page construction, LEGOMMI has a Zoom function which can be activated from the Options pulldown menu in the Page window. Available Zoom options are 100% (the default value), 200%, 300%, 400%.

 

Configuration: connections

Two types of connection are provided: between Iconreg objects in a regulation page; between different regulation pages.

1. Connection between Iconreg object

This operation links variables of LEGO modules acting on the variable-associated port. Every port will store the name of the connected port.

The graphical look of a connection is a polyline which links ports.

In order to create a connection the user first select "Connect mode" on the Option's pulldown menu and then, click on the source port and draw the line on the target port. This operation is made by clicking the first mouse button on the first port and by moving the mouse (with mouse button release) on the second; an angle on the connection line can be obtained by clicking the first button. The user can abort the operation by clicking with second mouse button (the connection in creation is deleted). When the mouse is over a port, it displays the tag (2 character) of the associated variable.

MMI also maintains connections when the user moves objects that are already connected each other.

Particular connections:

- Branched connections are prevented except when a dedicated elementary-module named "dira" is used;

- Loops inside a scheme are not accepted during compilation, and must be broken by the elementary-modules named "dtri", "dtrl".

1. Connection between pages (interface connections)

Connections between different regulation scheme require the following operations:

1) to close the pages

2) to click the "Interface Mode" on Interface Mode's pulldown menu

3) to open the interesting pages

4) to build the connection by clicking the concerned ports with the first mouse button.

5) to close the pages

6) to click "Interface Mode" on Interface Mode's pulldown menu to unselect this modality

This kind of connection is possible only between two "interface ports".

The connection is shown inside the interface ports by a little red rectangle. This kind of connection can be delete trying to delete the interface iconreg.

 

Configuration: data input

Every object has in the "Specific" a resource (named Input Value) containing all attempt variables and data values and descriptions of the associate module like an ASCII string. The characteristic of the form that will be used in I/O data values are retrieved from a file (form) which contains the list of tags and the description of all variables and data of the module. When the user works on a regulation scheme the form file is retrieved during the choice of the object (elementary regulation module).

The user can change the values by pressing "Input Value" and work on the window displayed. In fact, MMI displays two windows featuring the Tags of the block variable and the data retrieved once the user has chosen between variables or data on the a first dialog page.

Once the user has pressed "OK", MMI updates the values for the LEGO block associated which edited iconreg.
