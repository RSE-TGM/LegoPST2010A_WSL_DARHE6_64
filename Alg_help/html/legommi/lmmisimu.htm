LEGOMMI: synoptic animation

During simulation session the MMI system is composed by a pagination subsystem that activate/deactivate requested pages, requests the updating of the variables present on the visualized pages ( to the updating system) and act as a client in respect of run-time services. Simulation's variables are requested or send to the updating system which communicate (generally via ethernet) with the simulator.

Animation: MMI startup

MMI can start up with some already set pages by using the following parameters:

mmi -Context a.ctx -pag pg1 -pag pg2 -pagback pg2

-Context

sets a Context file different from the default Context.ctx.

-pag

sets a page which has to be open at MMI start-up on the local display.

-pagback

set a page characteristic; once the page is open, its default position will behind the other pages which already are or will be opened. This page is usually included among start up pages.

Display

During start-up MMI tries to open a connection with all the displays defined inside the context file and sends an error message on mmi main window in case of failure. Only errors in the local display connection can abort mmi. Every page can be opened only once for each display.

Limits

If mmi is used in the SCADA version, up to 10 pages can be open simultaneously while the figure rises to 100 with the simulator version.
