LEGOMMI: active objects

Active objects are a collection of graphics objects which have a pre-established dynamic behaviour. These objects can be used inside a synoptic page or included in active objects libraries. Starting from a series of basic objects contained in the "std.lib" library the user can personalize objects setting their resources (Resource Editor) or he can create new composite object by using the Composite container and instantiating inside it desired simple objects.

In the following "how to use" topics basic objects are grouped depending on their characteristic.

Objects: object handling

Page construction is completed after instantiation and a number of modifications to the active objects on the page have been performed.

 

Instantiation

- Instantiation. An object is generally retrieved from a library by clicking Add Icon's pulldown menu on the available active objects library. This operation displays the chosen library and the user, can retrieve an object by dragging it (with keeping the second mouse push button pressed) on a free place of the page drawing area.

 

Modification

- Selection. Generally, in order to manipulate one or more objects it is necessary to select them. However selection is not required for some operations such as moving and resizing. To select a single objects the user has to position the mouse pointer over it and to click the first button; selecting two or more objects at one time (inside the same page) is possible by pressing <ctrl> key simultaneously. In case of single selection, (without <ctrl> key) a new selection unselects the previous.

Several objects can be selected at the same time by drawing with the mouse a rectangle which contains the objects to be selected. This operation is made by setting the first corner with the first mouse button and then dragging the mouse to build the rectangle with the same button held down. To unselect an unwanted object click it before releasing the <ctrl> key.

- Resources. By using the Resource Editor the user can modify an already instantiated object. Some characteristics, typically the position, the dimension, the basic colors, can be modified through the Geometric resources class. Others can be modified through the available class resources of the object (such as Specific, Color attributes, Color change var).

- Moving. To move an object the user has to position the cursor in the center of the object, press and hold the second mouse button and drag the pointer to the drop site. Moving of several widgets simultaneously is permitted.

- Resizing. To resize an object the user has to position the cursor near one of the outer edges of the objects and press the second mouse button. The object will be surrounded by a grid which is divided in 9 areas. Every one allows only particularly resizing movement which is performed by dragging the mouse as shown by the arrows in the following figure. Object resizing is not permitted for "change page" and "iconreg" objects.

Editing

- Copy, cut, paste. This operation, applied to some selected objects, can be obtained from the popup menu of the page which is recalled by pressing the third mouse button. These operations involve both foreground and background.

LEGOMMI allows selective operations on foreground (objects) or background only after choosing the same copy, cut and paste buttons in background and foreground submenus.

- Duplicate. Duplication is different from copy only for the composite objects. A duplication of an object child of a composite, keep it as child of the composite. Not the same will happen in case of coping because the new object will be born child of the drawing area.

- Delete. One or more selected objects can be deleted by selecting the Delete option on the popup menu.

Objects: behaviour during animation

The behaviour of the animated objects is defined by the animation table defined as a group of resources for each color change variables. In LEGOMMI/Simulator version the table defines color and ranges limits (threshold) for the color change variable. In LEGOMMI/SCADA version defines colors for variable "states" defined by flags.

The user during animation has the possibility to insert perturbation and display variables trends using the AING and the PLOT push buttons displayed at the right side of every mmi's synoptic page.

This possibility is available for the objects which have almost the first color change variable, in particular:

Animated icon

Bitmap

Indicator

Iconreg

Dispreg.

In the same modality, once AING and PLOT will be clicked with the first mouse button a click on one of the above objects will display the Variable Editor.

In AING modality, if the first color change variable is set, Variable Editor will be displayed together with the Model and the Block fields already set. The variable set, as says above, is the first color-change variable but it is displayed only if it is an input one; in the others cases this field will appear blank.

Once the OK push button has been pressed, a defined perturbation window, called AING will be displayed.

The similar variable selection is given by mmi once the user has pressed the PLOT push button but differs from the above because Variable Editor does not filter only input but all variables: the first color change variable is always set (if defined).

Once the OK push button has been pressed, a window featuring a graphic object (Run time one variable type) will be displayed.

Before the user starts the two activity by pressing the OK push button can also modify the selected variable by means of the push button listed in the variable field.

During animation the user can display the connected page to the using one by clicking on the interface iconreg of interest. This operation opens the connected page (if it is closed) or popup it (if it is already open) and closes the current page.

By clicking on the drawing area of a graphic object (set in run time 1 variable configuration) with the first mouse button it is possible to modify the drawing variable using the Variable Editor. The graphic pop-up menu, displayed by clicking the second mouse button, is described in Graphic.

 

Objects: animated icon

By this term we define a rectangular window containing a user-defined draw with color animated parts (generally a schematic representation of a plant device).

The animated icon is connected with variables (animation variables) -that are of significance for the description of the device state or function-, calculated during simulation and with numerical constants that define the range of the variables. Variable values can cause a color modification in parts of the icon draw which the user, during draw construction, has defined as animated parts. Animated parts of a draw are obtained when the user draw a geometric elements using one of the enumerated color (1..4) defined in the color palette of the Draw Editor. Each enumerate color is associated with the corresponding animation variable and the color policy during run time is performed as defined by the color resources specified in the Color Change var. group.

Associated colors for each changing interval will define the icon's graphical behaviour.

Two types of animated icons are available.

Vectorial icon

Is a particular object which defines an animated draw in vectorial form, such as graphics command sequences which allow restoring. Permitted manipulations are:

Orientation change.

Dimension change.

Dimension change is not directly available; it is possible to modify dimension by changing the dimension of the internal draw (using Draw Editor).

The association of drawing by animated variables is contained in the vectorial draw resource which defines, in additions of the command that builds the drawing, the association of portions of its with animated variables values. These portions have been drawn by the user by using a color associated with a variable.

Bitmap icon

Is an object that uses the bitmap format and it is build through the standard Bitmap Editor. Only one color change variable is available and its change will affect the whole icon. The dimension is fixed and the resource "icon orientation" is not available.

Objects: indicator

The indicator is an object which can display the value of an associated variable through analogical or digital modalities.

The variable value is displayed after it has been filtered with the offset and scaling values have been defined in the resource as in the following relation:

out = SCALING*in + OFFSET

The number of digit displayed and the upper and the lower limits for variable's value are user defined.

N:B: If value is external to limits defined the maximum (or minimum) value is displayed.

Objects: iconreg

Iconreg is an object dedicated only to the regulation pages. In fact LEGOMMI prevents the instantiation of any kind of iconreg inside a synoptic page.

An iconreg object has been linked to a LEGO regulation module during iconreg construction (in library).

Some input and output ports are instantiated as children of iconreg and each linked with a module's variable for use in connection between objects. Depending on the regulation module associated, iconreg features an input form allowing initialization of LEGO module's variables and data values. When an iconreg is instantiated inside a regulation scheme a "Tag" is automatically generated for it's identification. The generated tag identifying the icon named is an acronym of 4 characters XXYY where:

XX is the icon progressive number

YY is the scheme progressive number.

Iconreg's symbol is a user defined vectorial draw animated by 4 variables (max) chosen among block's variable. The vectorial draw is included in an icon file. This file can be edited or created through the Draw Editor.

Iconreg generally has one or more of the following children:

1) Input and output ports for icon connection

2) Tags associated with the port (displayed on request)

3) Display representing the value of one of the LEGO module associated variables (dispreg)

An iconreg must be completely configured inside a library before any instantiation in pages. In particular MMI avoids:

1

to instantiate an iconreg without any associated form file

2

to instantiate any iconreg containing ports which are not associated with a LEGO variable

When an iconreg has been instantiated inside a page, an elementary regulation module is created for the regulation scheme. The elementary regulation block's acronym is assembled as follow:

&&&&XXYY

where &&&& is the elementary regulation module name.

The dimension of an iconreg is fixed by the associated icon which is imported or created through the Draw Editor. Other available resources can move the top, bottom, left and right border in order to obtain additive space for ports.

Iconreg can verify the compatibility between the module variable list (retrieved from form in memory) and the associated port variables. This control prevents error when the user changes the file name associated with the object in the resources after ports instantiating . In case of error MMI deletes ports

Information on iconreg resources is available in:

1 Iconreg layout and resourcelmi_aob_iconreg_ly

 

Objects: port

Port is an object dedicated only to the regulation schemes construction. It can be instantiated only on an IconReg object; then it is used for the construction of single components which will be insert inside regulation schemes.

The Port inserted in a regulation acts as I/O device; during the configuration of a regulation page, when the user choose Connect mode the port is sensitive to mouse click and start the drawing of a connection. The end of the connection it is realized when the user clicks on another port (destination port). A connection between ports means a connection between regulation variables: connected variable names are deduced from a particular resource of the port named "Associated variable name". This resource contains the name of one of the LEGO regulation module variable associated to the regulation icon.

Every port can be either input or output, digital or analogical, interface or not; ports will become red if analogical and blue if digital and two different geometrical symbol can be chosen by the user: arrow or square. Interface ports allows linking between variables of different schemes.

Connection are allowed only between same type ports (analogical, digital) and between complementary directions (input-output). The type Interface port is used inside interface icon and they allow connections between different regulation pages.

According to the set of "Set visual mode" on the Option pulldown menu in the window page, LEGOMMI changes the representation of a port: unmanage, manage, tag (displaying the connected tag).

Information on port resources is available in:

1 Port layout and resource 

Objects: dispreg

Dispreg is an object dedicated only to the regulation pages, it can be instantiated only into iconreg object during active object library creation.

Dispreg inherit the module name associated to their iconreg parent; it is also linked to one of the variables belonging to the module; it is used to show, during run time, the associated module name, the linked variable tag or its value in accordance with the choice made in "Set visual mode" on the Option pulldown menu in the window page. Naturally, during instantiation the user has to assign the linked variable name from the list of associated module variable.

Through Component and Subsystem LEGOMMI supplies a method to filter block selection.

By the "Component" resource the user can assign the editing icon to a particularly group of components such as Valves, Pumps, etc. When the filter Component is set this allocation will ease model block selection as the user is working with the Variable Editor . LEGOMMI works with an ASCII file named FileComponent.edf (local to the project directory) which contains all the project Components name.

The "Subsystem" resource is similar to the Component one except for its way of arrangement. Every subsystem group a part of a plant module i.e. a portion or the whole regulation, low pression devices etc. Once a particular subsystem has been set the user can filter in the Variable Editor the desired models. The ASCII file which gathers this information is named FileSubsystem.edf and must be local inside the simulator directory.

Another ASCII file named FileDefVarCC.edf is used by LEGOMMI to set a default color change variable name. In fact this file contains a list of rows featuring the names of a LEGO module and of one of its variables. During the assigning of color change variable the Variable Editor, once the user has chosen a particularly block of a module coded in the file, will display the associated block variable by default. Naturally the user can then chooses another variable.

By pressing Filename the user can import a file inside the icon. The imported file can be edited by the user, with relation both to the drawing and the animated behaviour, by assigning the color change variables and the related intervals. This operation can be made through the dedicated editor named "Draw".

DIGITAL INPUT

The user can change the associated variable through a window displayed by LEGOMMI which shows the list of available variables (input or output depending from the type port) belonging to the LEGO module referenced by its iconreg parent and the list of the already selected variables list.
