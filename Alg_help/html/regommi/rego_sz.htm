SETM regulation module

 

The regulation module SETM calculates the difference (UE error), in case multiplied by a gain K_ERRO, between the value considered for a quantity (US set-point) and its measured value IM, appropriately scaled down to make it comparable with the set-point (UM quantity).

The state of SETM module (automatic or manual), is defined by the logical variable J3 or by the value of logic related to the set-point J1. The US set-point can defines different quantities.

For clearness, it had better to discriminate among different cases, but first we point out some preliminary operations common to all of them.

The value scaled down from the measured quantity UM (comparable with US set-point) is calculated, dividing the measured quantity IM by its end scale value K_MISU (fixed by user):

UM = IM / K_MISU

Besides, the two output logical variables JE (met set-point increased) and JF (met set-point decreased) are put to the value "false".

1) SETM in automatic: J3 true.

User has to operate so that the output error UE is null, copying the scaled down measure UM on the set-point US:

US = UM

2) SETM in manual: J3 false.

2.1) Logic of set-point J1 true.

The US set-point is put equal to its TARGET II value:

US = II

2.2) Logic of set point J1 false.

2.2.1) Stationary condition: no calculation is carried out, therefore the US output keeps the value fixed by user during the filling phase of the file F14.DAT.

2.2.2) Transient condition: the two logical signals J4 (request of set-point increasing) and J5 (request of decreasing set-point) are compared. If they are equal (0 => no request, 1 => contemporary request of increase and decrease, are both ignored), for the US set-point the value of the preceding time interval (USP) is kept:

US = USP

If J4 and J5 are different, user tests which of them is "true".

a) J4 true (request of set-point increasing). The US set-point is increased of a quantity equal to the product of the time interval Dt by the KK gradient value:

US = USP + KK * Dt

the Je signal (increasing of met set-point) is put to the value "true".

b) J5 true (request of set-point decreasing). The US set-point is decreased of a quantity equal to the product of the time interval Dt by the KK gradient value:

US = USP - KK * Dt

the JF signal (decreasing of met set-point) is put to the value "true".

Finally we have the calculation operation of the UE error, which is common to all possibilities previously mentioned:

UE = K_ERRO * (US - UM)

where K_ERRO represents the SETM module gain.

During the initialization phase of SETM module variables, user must not pay particular attention.

In fact, since the value of IM measured quantity is scaled down dividing it by its end scale value K_MISU, user has to assign to this datum a value different from zero in order to avoid numerical problems (division with null denominator).

SOGA regulation module

 

The regulation module SOGA is a comparator (with or without hysteresis) between an input analogue signal and a reference gate; it provides as output (logical) a true or false value depending on whether the input signal exceeds or not the gate.

Splitting the case without hysteresis from those with hysteresis, we can explain more in detail the module functioning:

1) Without hysteresis (KL signal minor or equal to zero).

Defined as II the input signal and as KS the value of reference gate, the logical output JU is:

JU = false (or 0), for II < KS

JU = true (or 1), for II >= KS

Presenting by a diagram the transfer function, the graphic shown in figure 1 is obtained:

figure 1

2) With hysteresis (KL signal higher than zero).

In this case two gate values are available, (lower KSINF, upper KSSUP) which are calculated beginning from KS (reference gate) and KL (width of hysteresis):

KSINF = KS - KL

KSSUP = KS

Between the two calculated gates, only one is active (the other being irrelevant) which depends on previous history (state). Therefore, it is necessary to determine one or more variables which can univocally identify the system state; for the case in question it is sufficient to know the output value at the preceding time interval JUP.

If JUP is false, the active gate is the upper one KSSUP; defined as II the input variable, the output logical variable JU is:

JU = false (or 0), for II < KSSUP

JU = true (or 1), for II >= KSSUP

If JUP is true, the active gate is the lower one KSINF; defined as II the input variable, the output logical variable JU is:

JU = false (or 0), for II < KSINF

JU = true (or 1), for II >= KSINF

Presenting by a diagram the transfer function, the graphic shown in figure 2 is obtained:

figure 2

Note 1

Only during the calculation phase of a transient, it makes sense to use as discriminant variable the output at the preceding time interval JUP. On the contrary under stationary condition it is not possible to calculate the JU output in a univocal way. In fact we have:

JU = false (or 0), for II < KSINF

JU = true (or 1), for II >= KSINF

Within the interval included between the two gates, it is no possible to establish in a univocal way the JU value, therefore this is kept equal to the tentative value assigned by user.

The module SOGA has no data.

During the initialization phase of the SOGA module variables, user must not pay particular attention.

The regulation module SOGC is a comparator between an input analogue signal and two reference gates; it provides, as output (logical) a false or true value depending on whether the input signal is or not included within the interval identified by the two gates.

SOM2 regulation module

 

The regulation module SOM2 provides, as output, the addition, suitably weighted, of the two input analogue variables.

Once defined as I1 and I2 the two input analogue variables, as GUAD1 and GUAD2 the respective weights to assign to the two input and as UA the output analogue variables, in the formula we can write:

UA = GUAD1*I1 + GUAD2*I2

Note 1

It is evident that the SOM2 module can also carry out subtractions, for example it is enough to provide as weights the following values:

GUAD1 = 1

GUAD2 = -1

In this case the operation executed by the SOM2 module is the following:

UA = I1 - I2

SOM3 regulation module

 

The regulation module SOM3 provides, as output, the addition, suitably weighted, of the three input analogue variables.

Once defined as I1, I2 and I3 the three input analogue variables, as GUAD1, GUAD2 and GUAD3 the respective weights to assign to the three variables and as UA the output analogue variable, in the formula we can write:

UA = GUAD1*I1 + GUAD2*I2 + GUAD3*I3

Note 1

It is evident that the SOM3 module carries out algebraical additions if to some weights a negative value is assigned.

SWTC regulation module

 

The regulation module SWTC provides, as output, one of the two input analogue signals according to the value of the third input signal (analogue).

More precisely, once defined as I1 and I2 the two input analogue variables, as JJ the input logical variable and as UI the output analogue variable, in the formula we can write:

UI = I1 for JJ false (or 0)

UI = I2 for JJ true (or 1).

TIMF regulation module

 

The regulation module TIMF simulates the behaviour of a one-shot multivibrator sensitive to the rise fronts on the START input JS. More precisely, the logical output JU is normally positioned on the value "false"; it moves to the value "true", keeping this position for a time KT fixed in advance (TIMER duration), when the input JS changes over from "false" to "true" (or from 0 to 1 from which the term "rise front"), coming back successively to the value "false". It is interesting to note that such behaviour of the output is obtained independently from the variations that the input JS has during the wide time interval KT (figure 1). Therefore it is clear that the output signal JU, keeping "true" for a time interval fixed in advance, can serve as time basis in various applications.

 

figure 1

 

We can examine more in detail the operation carried out by this module.

First of all, user has to consider that "to count" the time, the variable ST is used. Then we identify the different possibilities according to the values of some logical variables:

1) Stationary condition: the output JU is put at the value "false" (a monostable has only one stable state, in this case "false").

2) Transient condition: using the input to the preceding time interval JSP, user has to consider if, in correspondence with the last 2 time instants (the preceding and the present), there are some "rise fronts" on JS.

2.1) Rise front on JS (JS "true" and JSP "false"). The output JU is forced to the value "true" and the counter is increased with a quantity equal to the time interval.

ST = ST + Dt

If the counter ST exceeds the interval KT fixed in advance, the output JU is brought back to the value "false" and the counter is set to zero, otherwise no action can be engaged in.

2.2) No rise front on JS.

2.2.1) Output signal JU false. No action can be engaged in.

2.2.2) Output signal JU true. The counter is increased with a quantity equal to the time interval:

ST = ST + Dt

If the counter ST exceeds the interval KT fixed in advance, the output JU is brought back to the value "false" and the counter is set to zero, otherwise no action can be engaged in.

TIMR regulation module

 

The regulation module TIMR simulates the behaviour of a one-shot multivibrator. More precisely, the logical output JU is normally positioned on the value "false" (input JS false); it moves to the value "true", keeping this position for a time KT fixed in advance (TIMER duration), when the input JS turn "true", coming back successively to the value "false". It is interesting to note that such behaviour of the output is obtained independently from the variations that the input JS has during the wide time interval KT (figure 1). Therefore it is clear that the output signal JU, keeping "true" for a time interval fixed in advance, can serve as time basis in various applications.

 

figure 1

 

Besides, as input, a RESET signal is present (prevailing), which, if "true" forces in any case the output JU to the value "false"(input of set to zero).

We can examine more in detail the operation carried out by this module.

First of all, user has to consider that "to count" the time, the variable ST is used. Then we identify the different possibilities according to the values of some logical variables:

1) RESET signal JR true. The output JU is forced to the value "false" and the counter ST is set to zero.

2) RESET signal JF false.

2.1) START signal JS true. The output JU is forced to the value "true" and the counter is increased with a quantity equal to the time interval:

ST = ST + Dt

If the counter ST exceeds the interval KT fixed in advance, the output JU is brought back to the value "false" and the counter is set to zero, otherwise no action can be engaged in.

2.2) START signal JS false.

2.2.1) Output signal JU false. No action can be engaged in.

2.2.2) Output signal JU true. The counter is increased with a quantity equal to the time interval:

ST = ST + Dt

If the counter ST exceeds the interval KT fixed in advance, the output JU is brought back to the value "false" and the counter is set to zero, otherwise no action can be engaged in.
