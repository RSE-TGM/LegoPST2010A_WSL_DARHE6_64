STEADY STATE ACTIVITY

The steady state activity calculates the steady state starting with values of variables and data defined in F14 file and produces the steady state file f24.dat having the same structure of a f14.dat.

Steady state: set computation parameters

 

The user can set the following parameters before calculating the steady state.

 

Tolerance:

Indicates a value for which the jacobian value is assumed as equal to 0 (steady state reached).

Default is 0.00001.

 

Mode:

Is the definition of the computation modality. In the "Direct computation modality" calculation stops automatically upon completion, while in the "Interactive computing modality" (not yet implemented) the steady state is calculated stopping at each iteration; next iteration is calculated when user press the step push button.

 

Initial system data/ Last computed data:

Specify where will be taken computation starting data, from the f14.dat or from f24.dat file (i.e. taken either from the edit data activity or from a previous steady state computation)

 

Steady state: run steady state calculation

 

Steady state calculation starts on activation of start push button. Information about jacobian calculations performed are displayed in the status window.

At calculation completion a message notify if steady state (accordingly with the tolerance value specified) has been reached.

 

Steady state: view calculation log file (lg3.out)

 

If convergence problems or errors due to starting values occurred an error message is displayed and the view of steady state log file (f03.out) is proposed: this file contains all messages written by write statements in LEGO modules' code and eventually error message written by H2O tables' functions (usually out of tables messages).

View of the log file is accessed also by selection of Print | Computing results on steady state main window.

From F03.out view window the user can print the file displayed (print push button).

 

 

Steady state: analyse calculation results

 

The user can look at the computation steady state results by selecting the Results push button from the Report's pulldown menu.

He can compare initial with calculated final variable values for all blocks (selecting from block's list the ALL MODEL BLOCKS item) or for a single block at a time (selecting a single block in the list).

Also geometrical and physical data can be displayed (although they are computation independent).

 

 

Steady state: change known unknown variable status

 

Known/Unknown editing allows users to manipulate the Known/Unknown status of model variables.

Users start the known/unknown edit program by activating the "Selecting unknown variables" choice from the "Actions" pulldown menu of the Steady state main window.

From the "nnEdit" main window users can operate in either of two modes: "Normal" and "Pairs." In "Normal" mode (the default) users manipulate variables - out-vars can be either marked or unmarked as known, and inp-vars can be either marked or unmarked as unknown. A star ('*') near variable long names indicates variable defined as known. In "Pairs" mode, users manage pairs - one known out-var and one unknown inp-var. A hyphen ('-') near variable short names indicates variable in a pair.

Above manipulations just enable a loop "edit known/unknown status" and "perform a steady state calculation".

Auxiliary dialogs are also available to:

a) filter a subset of model variables to operate on,

b) edit variable data,

c) graphically inspect current known/unknown status.

 

Information on change status working pages is available in:

1 Change known/unknown status, main page

2 Change known/unknown status, filter page

3 Change known/unknown status, graphical auxiliary pages

4 Change known/unknown status, value page

 

 

Steady state: analyse diagnostics - unsatisfied equations

 

Analysis starts from (step 1) the view of the number of times each equation was unsatisfied. The list is ordered by decreasing no. of times the equation cannot satisfy tolerance value.

 

Selecting an equation a second dialog (step 2) is visualized: it contains:

a) additional equation information: block owning the equation, order number of the equation (block's relative), descriptions and units of measurements of the equation.

b) list of all iterations and corresponding residuals calculated for the specific equation.

 

Iteration selection from list leads to step 3: a further dialog displays block name and description; a list below displays variables' names, values and jacobian matrix values in the specific iteration and equation. A star ('*') near variable name indicates variable defined as known.

 

Steady state: analyse diagnostics - iterations history

 

Analysis starts from (step 1) the view of a summary of iteration history displaying for each iteration if there were jacobian matrix recalculation or convergence problems: a star ('*') in the relative column indicates a jacobian recalculation or convergence problem.

For each iteration are also displayed max. calculated residual, the norm , the no. of not satisfied equations.

 

After iteration selection (step 2) a list of all not satisfied equations ordered by decreasing residuals, residuals (normalized and not normalized value), unit of measurement are shown for the specific iteration.

 

Equation selection leads to step 3: a further dialog displays block name and description; a list below displays variables' names, values and jacobian matrix values in the specific iteration and equation. A star ('*') near variable name indicates variable defined as known.

 

Steady state: analyse diagnostics: jacobian matrix analysis

 

Analysis is based on the view (of all or part) of jacobian matrix at a chosen iteration.

User first select iteration no. from a list showing jacobian matrix recalculation and/or singular matrix (step1): the following dialog pulldown menu selections are enabled (N.B. all information displayed refers to selected iteration):

Investigation | Blocks: generates a dialog displaying the model block list and a scrolled window containing part (or all) the jacobian matrix. Setting a block, jacobian matrix rows and column belonging to block selected are shown; if 'ALL MODEL BLOCKS' item is selected from list the whole jacobian matrix is displayed.

Investigation | Unknown variables: generates a dialog containing two lists: Unknown variables list and jacobian matrix column list. Selecting a variable in the first list, the corresponding jacobian matrix column is shown in the second list.

Investigation | Singularity: the submenu is enabled only if at the iteration selected jacobian matrix is singular.

Investigation | Singularity | Information: displays information about the type of singularity (structural or due to coefficients).

Investigation | Singularity | Null rows: displays all null rows at iteration selected. Suggestions for problem solution are proposed.

Investigation | Singularity | Null columns: display unknown variables which correspond to null columns of jacobian matrix. Suggestions for problem solution are proposed.

Steady state layout: diagnostics pages

 

Layout of diagnostic pages is based on scrolled lists displaying iteration, equations and jacobian matrix related information. See how to section for a functional description.

 

Navigation into different diagnostic pages is allowed by item selection. Sometimes selection is obtained with double clocking (however a message specify that double clicking is needed).
