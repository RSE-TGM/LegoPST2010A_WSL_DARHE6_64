MODULE

An object class modelizing a physical component behaviour (a pump, a regulator, a boolean operator, ...) in terms of input and output variables names, data names and dynamic behaviour. Some characteristic of the object (such as names and number of variables) depends on parameter specified by the user during instantiation.

 

BLOCK

Is the instance of a module which is a typical component of the described plant.

The block is completely defined in terms of input and output variables (names, type and number), physical and geometrical data names and dynamic behaviour. The block configuration can proceed with the definition of initial variables values and data values.

 

MACROBLOCK

Is a symbolic icon containing the icon block of a model

 

PROCESS LIBRARY

The collection of standard and user made process modules (libut). The library contains for each module a fortran file describing module structure and behaviour, bitmap file (used for graphic icon representation) and module related documentation (ASCII text file). Depending on context the term process library is also used indicating only fortran modules library.

 

MODEL

A subsystem of a power plant simulator constituting an homogeneous unit. It is possible to have more than one model of the same subsystem to study alternative approach at the problem. Every model is made using the appropriate module library.

 

MODEL DIRECTORY

Is the place in which the information necessary to build a model and to run it stand alone are stored.

 

ICON

Is a graphic representation that can be associated to one LEGOCAD module. An icon can not be associate to more than one module.

Every icon is graphically represented by a rectangle area containing the module name and an area featuring a graphic symbol stored in bitmap format. The icon size can vary with the minimum being established by the smallest rectangle able to contain the name.

A standard icon is the default symbol for all modules which have not been associated to any icons; this is also the starting icon for the creation of a new one.

 

GRAPHICS OBJECT SELECTIONS

A graphic object can be selected by clicking the mouse left button when the pointer is inside the icon. The icon will be surrounded by a red frame. Multiple selections are possible by pressing simultaneously <ctrl> on the keyboard and selecting further icons. An alternative multiple selection is rubberbanding.

Likewise a name in a list (variable, model or module name) will be select by pressing the mouse left button inside the row. The name will reverse.

 

GRAPHIC OBJECT DESELECTIONS

Can be performed by clicking left mouse button in a point where there is no object. A single selection also deselects previous selection.

 

RUBBERBANDING

Simultaneous multiple selections are possible by drawing a rectangle obtained by dragging the mouse with the left button pushed.

 

CONTROL BLOCKS LIBRARY (libreg)

Is a collection of standard blocks representing automation primitives used to build control schemes.

 

CONTROL SCHEMES LIBRARY (libut_reg)

Is a collection of user made modules representing automation schemes used to build automation models.

 

VARIABLE OF LEGOCAD

The LEGOCAD variable is a software variable (defined in a LEGOCAD module) which is used in the module with a specific meaning to define a physical quantity of the component modelled. Thus there are input, output and state variables.

 

S01

S01: is the simulator topology definition file. It contains the list of the models used, and their interconnections. It also contains some other data useful for run-time simulation.

 

SCHEME

Is a set of interconnected automation modules representing a well defined automation subsystem.

 

DATA OF LEGOCAD

Is a collection of physical and geometrical data used by LEGO and defined in the modules.

 

SIMULATOR

Is a set of models linked together. If the level of detail is very low the simulator could contain only one model; often it contains two or more models. There are no differences in executing a single or multi models simulator. It is possible to have more than one plant in the same project to study alternative approaches or to track its growth.

 

SIMULATOR DIRECTORY

Is the place in which the information necessary to build a plant and to run all the models linked in it are stored. The run-time tool is the same used to run the single model.

DIAGRAM PAGE'S INDEX

This window display pages that compose the whole diagram as labeled boxes. Pages may represent a logical subdivision of the plant.

From page's index you can select and open pages that describe the plant part using a schematic diagram.

 

MODEL DIAGRAM PAGE

Model diagram page contains a simple graphic description of the plant.

It is composed by icons representing LEGO blocks, comments labels, connections (in general arrows) and graphical symbols.

The diagram describe the internal structure of the model and constitute an useful worksheet during topology definition and model data editing. LEGO blocks are directly selectable from diagram page and specific operations on selected blocks enabled.

 

BLOCK'S INSTANTATION

When a model has already been created the user can add new blocks in the model through instantiation.

To instantiate a new block two different windows related to select operating mode are displayed but the logical steps for the instantiation are the same:

* select module from a list

* assign block name and description

The instantiate windows depends on the selected operating mode.

 

COLLIMATION

Is an utility of the GRAFICS tool (enable by the Defaults/On/Off Collimation pulldown menu) which allows, to read variables values by dragging the mouse cursor on the draw.

 

F14.DAT

Is the model data definition file (ASCII format). It contains variables' values, physical and geometrical data used as attempt values for steady state calculation. It also contains the specification of which variables are to be considered as known terms in equations resolution.

 

F01.DAT

Is the model topology definition file (ASCII format). It contains a list of the used modules, their identification tags (blocks' names) and their interconnections.

 

F22.DAT

It contains the values of a user defined set of variables sampled and recorded during simulation (binary format). Variables recorded can be selected by user before starting simulation.

 

FORAUS.FOR

Is the user defined fortran subroutines file. It is the only way for the user to customize the behaviour of the process modules. Only some aspects of a module can be modified.

 

FIND AREA

Is a portion of window which contains a few objects to find a name (names of blocks, models, variables etc) in a list. These objects consist in a text field where the user can insert the name searched for and two arrow buttons to choose the direction of the search in the list.
