MODULE MART CARD A

Model Generalities

 

TURBINE SHAFT ROTATION SPEED MODEL

 

When generation units forming an electric power production plant, supplies the central station auxiliaries, unit itself becomes a limited power network (insulated network). Such a situation occurs during a load reject transient.

It began when main switch gear connecting the a.c. generator to external main network cuts off. So, in this case, resisting load consists of central station auxiliaries keeping connected, through an auxiliary transformer, with the a.c. bus-bars. Unbalance which occurs between fed mechanical power and required power, considerably lower (for a steam plant about 5% of the nominal power), causes a quick increasing in rotational speed that has to be controlled and limited.

Rotational speed successively decreases because of power dissipated through steam turbines in lamination, ventilation (both of turbine and a.c. generator), friction and, in case of gas-turbine units, the power absorbed by compressors. In fact during this type of transient gas-turbine units are supplied with a fuel flow rate such as to guarantee flame keeping in th e combustion furnace but not the unit self- maintenance. So power absorbed by compressor is higher than power supplied by the turbine determining a gas-turbine deceleration.

For gas turbine units, it is present an "external starting torque" input variable, to allow the simulation of the related starter subsystem (mechanical or electrical).

Referred to an electric power production unit, consisting of 'n' (max.9) generation units (both steam turbines or gas-turbines units), the MART module simulates the dynamics of rotational speed in unit machines. In the module is assumed that electromechanical oscillations of machine shafts, integral each other, is negligible.

The mathematical model through the simulation is carried out, essentially consists of rotating mass equation repeated for 'n' generation units:

+ QAVV(i) i = 1, ... n (1)

where:

E(i) = total kinetic energy of the i-th generation unit

Q(i) = total power supplied by the i-th unit

QAC(i) = coupling power related to the i-th turbine (see note 1)

QUAS(i) = power utilized in supplying the part of auxiliaries related to the i-th unit (see note 2)

Qd(i) = total power dissipated by the i-th unit (see note 3)

QAVV(i) = power given by the external starter to the i-th unit (see note 4)

 

Note 1

QAC(i) is a coupling term of several turbines installed in a power production unit. It takes into account the transferred or absorbed power by the i-th turbine because of the different rotational speed acquired as regards to average speed of remaining n-1 turbines.

In the module the encoded expression of QAC(i) term is:

QAC(i) = COP * (OM(i) - OMRET(i)) i = 1, ... n (2)

where:

- COP = coupling coefficient

- OM(i) = speed of the i-th turbine

- OMRET(i) = mean speed of remaining n-1 turbines

To the coupling coefficient COP, having torque dimensions, no analytical expression is given. It is assumed as constant and equal to 10 MW/(runs per minute). Such value is obtained so that, during the more significant transients, speed curves referred to the single units meet the actual ones. In particular is imposed that in case of load rejection (the most significant transient), curves speed referred to different units are identical.

Note 4

QAVV(i) takes into account the power given by the external starting unit of the single turbine. It has to be used for gas-turbine units to allow the beginning of shaft rotation when the machine is halted. The QAVV(i) term is taken into account only when the i-th unit is in starting state [when AVV(i) = 1.].

PHYSICAL MEANING OF MODULE INPUT-OUTPUT VARIABLES

 

[rpm] RPMi rotational speed of the i-th turbine

[W] QTUi power supplied by the i-th turbine

[ ] ASCi trip button for the i-th turbine (see note 2)

[ ] AAVi start button for the i-th turbine (see note 2)

[N] STTi torque given to the i-th unit by the external starter (see note 1)

[ ] ARIF load rejection button (see note 2)

[W] QAUS power absorbed by the auxiliary systems

[rpm] RPMI main frequency of network which unit is connected (see note 2)

 

Note 1

If AAVi < 0.9 (i-th turbine is not in starting state) the external torque (STTi value) is neglected.

 

Note 2

If RIFC < 0.1 and simultaneously for all turbines SCA(i) < 0.9 (i = 1, ..., n) (load rejection occurs and none turbine is tripped) no calculation is made and to the rotational speed ROM(i) (i = 1, ..., n) the value of network main frequency RPMI [RPM(j) = RPMI j = 1, ..., 1-1, i+1, ..., n] is assigned.

It is useful to note there is no restriction about the number of turbines which may trip. So, as extreme possibility, simultaneous tripping of all the 'n' turbines can be simulated.

Where RIFC > 0.1 (load rejection condition), the rotational speeds RPM(i) are determined through the solution of rotating masses equations mentioned above.

 

 

MODULE MART CARD C

Physical and Geometrical Data

 

In addition to initial values of the MART module input-output variables (meanings defined above), user must assign the following data:

[s] TAVVIA(i) starting time of the i-th turbine (i = 1, ..., n) (see note 1)

[W] POTNOM(i) nominal power supplied by the i-th turbine

[rpm] OMNOMI(i) nominal rotational speed of the i-th turbine

[p.u.] QVENTI(i) ratio (under nominal conditions) between power dissipated through ventilation and power supplied by the i-th turbine (see note 2)

[p.u.] QLAMIN(i) ratio (in nominal conditions) between power dissipated through lamination and power supplied by the i-th turbine (see note 2)

[p.u.] QATTRI(i) ratio (under nominal conditions) between power dissipated through friction and power supplied by the i-th turbine (see note 2)

[s] TLOAD closing mean time of the valves feeding plant turbines (usual TLOAD = 0.3 s).

MODULE MART CARD F

Residuals

 

The module mathematical model consisting of the equation mentioned in first paragraph gives rise to the following residuals:

RNI(i) = RPM(i) - OMEGC(i) i = 1, ..., n

where OMEGC(i) assumes different expressions in accordance with the following possibilities:

1) if neither load rejection nor turbine tripping occur:

OMEGC(i) = RPMI i = 1, ..., n

2) if there is not load rejection but the i-th turbine is tripping, for the last the OMEGC(i) is deduced through kinetic energy expression:

E(i) = ½ * I(i) * OMEGC(i)² i = 1, ..., n

while for the remaining ones:

OMEGC(j) = RPMI j = 1,... i-1, i+1, ..., n

3) if there is load rejection, OMEGC(i) is deduced through kinetic energy expression:

E(i) = ½ * I(i) * OMEGC(i)² i = 1, ..., n

The residual is normalized as regards to an angular speed of 300 runs per minute.
