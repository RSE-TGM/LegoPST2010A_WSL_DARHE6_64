MODULE GIPO CARD A

Model Generalities

 

HYDROMECHANIC COUPLING

 

The GIPO module simulates the coupling between a motor and a machine tool. It is used together with the MAS3 module (asynchronous motor) and to modules simulating the operation of machine tools: typically CPOM (centrifugal pump) and VENT (fan).

The GIPO module considers three typical coupling cases:

 

Case (A) Motor coupled with machine tool through an HYDRAULIC JOINT and directly coupled at the same time with another machine tool (it is the case of the PUMP BOOSTER+MOTOR+SUPPLY PUMP assembly).

The booster pump rotates at the motor speed while the supply pump can vary its speed according to the value assumed by the input variable ALSC simulating the operation of the pump "soop".

 

Case (B) Motor coupled with pump through a speed multiplier (reducing gear).

 

Case (C) Motor coupled with fan through a speed multiplier (reducing gear).

With respect to Case (B) the resisting power expression depending on the ALSC input variable changes. It represents the position of gates on suction.

 

In the cases concerning pumps the RESISTING POWER is assumed to be a function of SPEED and FLOW RATE acording to an imposed law.

For fans, the RESISTING POWER is assumed to be a function of SPEED, FLOW RATE, OPENING of gates according to an imposed law.

 

The module receives at input:

 

- the active power QASS supplied on axis by motor

- the mass capacity WPOM of the machine tool

- the machine tool fluid density DENS

- the ALSC position(of scoop if pump, of gates if fan)

 

The module supplies at output:

 

- the angular speed OMMO of the electric motor

- the angular speed OMPM of the machine tool

 

 

In the mathematical model adopted the following hypothesys are assumed:

 

it is assumed that power dissipated for mechanical losses is directly proportional to the speed, in particular at rated speed that is 1 % of the rated power

in steady state the speed at the joint output has a linear trend as the scoop varies

in Case (A) the ideal joint is assumed, in all the case it the ideal multiplier/reducing gear (unit yield) is assumed

it is assumed that the motor dynamics reflects on the dynamics of the machine tools, passing thouth the multiplier in directly proportional manner

it is assumed that the motor dynamics reflects on the dynamics of the machine tools, passing thouth the hydraulic joint in directly proportional manner (IRREALISTIC HYPOTHESIS TO BE CANCELLED).

 

The module is based on the following equations:

 

[1] equation describing the variation of rotation kinetic energy of a system according to the motive and resisting power applied. It is a differential equation solved in the algebraic mode, considering in the transient state of the value of speed at the previous time step.

 

[2] equation expressing the link between the speed of the machine tool and the speed of the electric motor (algebraic equation).

RELATIONS USED INSIDE THE MODULE

 

OMRIF = OMBA (reference angular speed in rad/s)

GFUN = FATMOL*VELZERO/VELNOM1*(1 + ALSC*(VELCENT/VELZERO - 1))

GFUN is the ratio between the supply pump speed and the motor one.

GIN = OMRIF*OMRIF*(FINERZ2 + FINERZ1/(GFUN**2))

GIN is the equivalent inertia referred to the speed of the machine tool.

 

Note 1 in the case of coupling with MULTIPLIER: ===> GFUN = FATMOL

 

The following is determined for the different cases, on the basis of data and inputs:

QRES (resisting power)

QPERD (dissipated power)

 

and from these:

QRESTOT = QRES + QPERD

MODULE GIPO CARD F

Residuals

 

The equations forming the module mathematical model provide the following residuals (the first one applicable to transient state):

 

RNI(1) = (GIN*ABS(OMPM)*(OMPM-OMPMN1) -

DTINT*(QASS*W0*H0 -QRESTOT))/POTNOM

 

RNI(2) = OMPM - GFUN*OMMO

 

Where OMPMN1 indicates the speed at the previous iteration and DTINT indicates the integration interval; while the remaining variables have the physical meaning already defined in cards B and C.

 

 

Note 1 When the system is in steady state:

if OMPM <= 0 RNI(1) = OMPM

if OMPM > 0 RNI(1) = (QIPM*W0*H0 - QRESTOT)/POTNOM
