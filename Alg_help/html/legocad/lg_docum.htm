DOCUMENTATION ACTIVITY

 

The documentation activity allows to print and insert model description information.

Documentation provides a series of dialogs for a guided insertion of information of interest for model description and generates a formatted and standardized report.

 

HOW TO

1 Insert block or model notes

2 Insert data or variable information

3 Show and print documentation

 

DOCUMENTATION WINDOWS LAYOUT

1 Documentation main window

2 Notes page

3 Data and variables pages

4 Print preview page

5 Unit of measurement page

6 Models variant page

 

Documentation: Insert block or model notes

 

The user can insert some notes about the model in use or a selected block through the notes page which is called by pressing the "Note" push button.

This push button is available either in the main window documentation page (model notes push button) or in the data/variables pages (block notes). In particular, "geometrical data of the block" and the "in/out variables of the block" pages are available after the block has been selected in the list.

 

Information on the pages layout is available in:

1 Documentation main window layout

2 Notes page layout

3 Data and variables page layout

 

Documentation: Data or variables added information

 

The user can insert further information about geometrical data by calling the "geometrical data of the block" page and about input/output variables through the "in/out variables of the block" page.

These pages show for every item (datum or variable) the symbol, value and unit of measurements.

Added available fields are "Data source", "Rev" (revision), "Page" (inside the document) and "Date"(of the modification).

In "geometrical data of the block" page is available (for each datum) a push button to insert enumerated block's notes.

 

Each window displays the "Note" push button to call the notes page for the block.

 

Information on the page layout is available in

1 Documentation main window layout

2 Notes page layout

3 Data and variables page layout

 

Documentation: Print notes

 

The user can see and print any part of the documentation by selecting Print's pulldown menu. Three choices are available: data documentation, variable documentation, steady state documentation although the same "print preview page" will always be displayed.

 

Documentation layout: main window

 

Besides a pulldown menu the documentation main window has a scrolled window containing the list of the model blocks. Every row contains the block name, the module name and the block description.

A push button named "Note" displays the note page to insert block or model notes.

Documentation layout: notes page

 

The notes page features a label bearing the model's name and a scrolled window where the user can insert his remarks.

Confirmation buttons ("OK", "Apply", "Cancel") are displayed.

 

Documentation layout: data and variables pages layout

 

Both pages have description and confirmation areas but differ as to the displaying data area (scrolled window).

 

Description area:

featuring the "Note" push button which calls the notes page when clicked as well as the name and the block description labels.

 

Confirmation area:

featuring the following push buttons: "OK" to store and close the page, "Apply" to store the page and "Cancel" to close the page.

 

Displayed data area of the "geometrical data of the block" page:

all data are displayed in a row with the following text fields :

Name, Value, Data source, Rev, Page, Date, Note.

There is also a push button to insert a numbered note.

 

Displayed data area of the "in/out variables of the block" pages:

all data are displayed in a row with the following text fields :

Name, Value, Unit, Descr (push button), Data source, Rev, Page, Date.

 

Documentation layout: print preview page

 

The print preview page has the following areas:

 

Selection area:

featuring an option menu to select the index or the documentation page; two arrow buttons; two text fields showing the current page and the total number of pages; a "Goto" push button to select the number of the page which will be loaded. This last push button can display a simple "change page".

 

Scrolled window area

featuring the documentation text

 

Selections area

featuring the "Print" and the "Cancel" push buttons. A click on the "Print" push button will display the "print documentation" page which can configure the model to be printed.

Documentation layout: models variant page

 

The user can see the different model's steady states that have been saved through the "model's variant" page. This page shows all the steady state data files.

The page has the following areas :

Data area:

featuring the identifier of the steady state, the file path name and a select push button.

 

Confirmation area

featuring two push buttons one to close the page ("Done") and one to display the selected steady state file ("View").
