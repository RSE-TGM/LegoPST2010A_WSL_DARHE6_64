TOPOLOGY ACTIVITY

 

This activity constitute the first step in model building.

The user create or modify the model internal structure using elementary modules from the process module library, instantiating them in the model (instantiated modules are called blocks).

Linkage between LEGO blocks are made connecting blocks' variables using the Variable Editor.

During topology activity the user can instantiate new blocks, connect variables, delete blocks or connections.

The build up of the topologic scheme can be performed according to two modalities: Symbolic diagram (with the help of graphics) and Only by list.

2 Define filters

3 Use only by list modality

5 Modify a model that already exists

6 Instantiate new blocks

Topology: symbolic diagram

 

In this modality the user can use a schematic representation of the plant as a worksheet for the construction/modification of the model structure. The model can be divided in multiple pages. The single page represents part of the model in terms of constituting blocks (shown as icons); using arrows, graphics symbols and labels the user can enhance information provided by the diagram.

Operations on blocks (such as blocks delete, modify description) are enabled when icons are selected. Selection can be performed in a graphical way (on the diagram page; graphic selection) or by selection in the scrolled list of blocks constituting the model (on topology editor page).

 

Topology activity shows two windows; see topology editor and diagram page's index for their layout. Through the diagram page's index is possible to open another window, i.e. model diagram page.

 

Connection of variables is made by user selecting two blocks (icons in the diagram page or items in model blocks list on topology editor page) and then opening their variable editors.

Topology: define filters

 

Filters are a series of strings of max. 4 char. used for selectively visualize only a group of input (or output) variables using variable editor.

Filter editor is activated selecting main window's pulldown menu button Option | Define Filter.

The user can specify a filter name (that will be used as a label in variable editor Filter option menu) and specify up to 10 filter strings.

During Variable Editor filtering, are displayed only variables' names whose first n characters are equal to the n characters of one of the filter strings (where n is filter string's length). If you select a filter in Variable Editor's option menu input and output variables lists are updated accordingly (only variables accepted by the filter are displayed).

From main window's pulldown menu selection Option | Default filter the user can select which filter, from existing ones, may be applied when variable editor is called.

 

Topology: only by list

 

This modality creates topology schemes without the help of graphics using only model blocks's list.

This modality display only the topology editor page

The researcher can connect blocks after selecting than in the topology editor page list by opening variable editors

1 Topology editor layout

2 Variable editor page layout

 

Topology: variable editor

 

In order to connect variables related to two blocks, two variable editor (one for each block) windows must be opened. Variable editor must be opened when the block of interest is selected.

 

Variable editor shows two lists displaying block's input and output variables. If variables are connected a lock icon is displayed near the variable name.

 

Internal variable editor option allows the user to examine the name of corresponding connected variables of the other block (View menu) displayed directly below each name of the connected variables of edited block.

An option menu is also available to filter variable lists (Filter option menu) accordingly with user defined filters.

 

The connection is made by dragging (with mouse button 2) an input (or output) variable of a block and dropping it on an output (or input) variable of the other block.

Besides, LEGOCAD prevents incorrect operations such as more than one connection for every input variable, connections of variables belonging to the same block, connections of input/input and output/output variables.

Not homogeneous variable connections cause a warning.

 

Deleting a connection involves the following steps:

1 Select show connections option.

2 Select connected variable (of the other block; displayed below edited block's variable).

3 Delete connection (from Edit menu of Variable Editor)

 

Information on the page layout is available in:

 

1 Variable editor page layout

 

Topology: modify a model

 

To modify an already existing model the user can work either with symbolic diagram modality or with the only by list modality.

If the researcher wants to work with a block it is necessary to select the block first.

After selection it is possible to modify or create new connections (see variable editor), to delete the block and to modify its description.

N.B: deletion of blocks in enabled only when no Variable Editors are opened.

Instantiation of a new block is always permitted.

 

Topology: block's instantiation

 

When a model has already been created the user can add new blocks in the model through instantiation.

To instantiate a new block two different windows related to select operating mode are displayed but the logical steps for the instantiation are the same:

* select module from a list

* assign block name and description

If symbolic diagram modality is selected one of the icons representing the module must be dragged and dropped (with second mouse button) on the diagram page window in the desired position.

To complete module instantiation for some LEGO modules you are asked for additional information (an instantiation dialog box pop-ups automatically). A question and answer sequence managed directly by the LEGO module is displayed; question are displayed into a scrolled text widget; answer must be edited in the text field below and confirmed activating the send button.

The instantiation windows layout are :

 

1 Instantiation window in symbolic diagram modality

2 Instantiation window in only by list modality
