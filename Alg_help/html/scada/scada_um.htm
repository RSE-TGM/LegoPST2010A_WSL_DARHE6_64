SCADA interface 

SCADA is designed to work in two different ways:

character interface

graphic interface

To have SCADA running with the graphic interface it’s necessary to compile it having configured X_USR_INTERFACE environment variable to 1, while to have SCADA running with the character interface X_USR_INTERFACE must be set to 0.

 

Character interface

Character interface allows machine maximum resource saving and it is very rudimentary.

On the display it appears:

- SCADA -

Q) Quit

By pressing ‘Q’ the program ends.

 

Graphic interface

DISPLAY (if the SCADA was generated with the <XUSR_INTERFACE> flag OFFSET_SCADA_PORT: this variable defines the offset added by SCADA to assigned port numbers for its services (connection with MMI stations, connection with the simulator [data and command] and with the configuration station).

How to execute SCADA

In order to start the SCADA process you have to type <scada> at the prompt command line.

SCADA has several starting flags: Type <scada /h> to have more information about the starting flags; on the terminal this message will appear:

 

SCADA

Use: SCADA [/Ffunction] [/Alarm_option] (/F function - /A allarm options)

[/Tcycle] ( /T time of one cycle)

[/Vlowest_rate] [/Mmmi_number] (/V the lowest refresh rate of MMI’s page

/M how many MMI you can connect to the SCADA system) function:

T logs enabled

A archive enabled

alarm options: R single alarm acknowledge enabled

S alarm acknowledge log enabled

V alarm browsing enabled

C alarm account for database and zone enabled

W closing weekly logs

F alarm string format: HH/MM/SS.

Lowest_rate: the lowest refresh rate of MMI’s page (the default is 1 Hz the max is 2 Hz)

Mmi_number: how many MMI you can connect to the SCADA system (a number between 1 and 16, the default is 1)

 

A possible starting macro is:

scada /FAT /ARV /M2

In this way logs and archives are enabled with /FAT, single alarm acknowledge and alarm browsing are enabled with /ARV and it is possible to connect to two mmi.

It is recommended, for two reasons, to activate SCADA via the starter process as you can see in the following command line:

starter /usr/users/scada/pscs32/bin scada /FAT /ARV /M2

<process path> <process name> <process flags>

 

The starter process is able to rerun SCADA if it has an abnormal termination. Furthermore starter is owned by root (having the "s" flag set and its children (like SCADA) inherit this feature.

In this way the SCADA process is allowed to alter its user-id (in order to gain root authority) to manage its threads priority index.
