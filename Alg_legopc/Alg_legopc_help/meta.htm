MODULE META

CARD A

Model Generalities

CYLINDRICAL THIN METAL WALL

 

In the META module is represented the dynamics of the temperature in a thin metallic wall with cylindrical geometry, as a function of the exchanged thermal power at the boundary surfaces (internal and external).

The metallic wall whose behaviour has to be simulate, is subdivided in an arbitrary number of cells. Each of them is characterized by a given exchange surface and a certain volume of metal. Inside the cell heat transfer occurs by conduction in radial sense. The longitudinal conduction phenomenon is not considered. As hypothesis it is assumed that the wall has a thin thickness in order to adopt a representation in single radial cells.

For each cell the following equations are written:

1) conservation equation of metal energy

2) equation defining the dependence between the metal external wall temperature and the mean temperature

3) equation defining the dependence between the metal internal wall temperature and the mean temperature.

The first equation is differential and defines (along the time) the variation of metal mean temperature as a function of exchange thermal powers. The equation 2) and 3) are algebraical and represent (under stationary condition) the solution of the Fourier's equation for the considered geometry.

For each cell the module calculates the following variables:

1) metal medium temperature

2) external wall temperature

3) internal wall temperature.

The META module can be easily be used (together other ones included in the LEGO library) in building mathematical models related to plants components.

As example, an evaporator for fossil fuel boiler can be built utilizing the following modules:

CCMB or CCDY + META + EVAF ==> combustion furnace + walls + fluid

MODULE META

CARD C

Physical and Geometrical Data

 

In addition to initial values of META module input and output variables (meanings defined above), user must assign the following data to complete the file F14:

Data needed to define which sign is used to calculate the modules exchanged powers having in common the metallic wall.

[ ] IPO1
if no data is written ==> power exchanged with internal side of the cylindrical wall is calculated in accordance with the expression:
Qi = K*S*(TPi-TFi)
that is > 0. if internal fluid takes heat

else if IPO1 -1. ==> power exchanged with internal side of the cylindrical wall is calculated in accordance with the expression:
Qi = K*S*(TFi-TPi)
that is > 0. if internal fluid gives up heat
[ ] IPO2
if no data is written ==> power exchanged with external side of the cylindrical wall is calculated in accordance with the expression:
Qe = K*S*(TFe-TPe)
that is > 0. if internal fluid gives up heat

else if IPO2 -1. ==> power exchanged with external side of the cylindrical wall is calculated in accordance with the expression:
Qe = K*S*(TPe-TFe)
that is > 0. if internal fluid takes heat
[kg/m³] ROM metal density
[K/kg/C] CM metal specific heat
[W/m/C] COND metal conductivity

If metal conductivity depends on temperature, it needs to define the subroutine COMETA(ICO, TM, CON) (see the section regarding user subprograms).

Data needed to define the cylindrical wall geometry

Each cell is defined through the following data:

[ ]

NTUB

number of cylindrical metal pipes in parallel configuration

[m]

DIAI

internal diameter of single pipe

[m]

DIAE

external diameter of single pipe

[m]

LUNG

cell length.

When geometrical data concerning several cells are equal, it is enough to assign this data to the first cell leaving blank the fields related to other cells.

MODULE META

CARD E 

User's Subprograms

If metal conductivity depends on temperature, user must provide the following subroutine:

SUBROUTINE COMETA(ICO, T, COND)

Meaning of FORTRAN's symbols:

[ ]

ICO

pointer (integer value) in input to the subroutine which user can utilize to chose the function COND = F(T) among a "menu" of funtions assembled in the subroutine itself

[K]

T

wall temperature provided in input to the subroutine

[W/m/C]

COND

conductivity calculated by the subroutine COMETA

E.g.: To assign the steel conductivity defined in the user subroutine COMETA, the following data has to be assigned in the file F14 for META module which deals with the steel wall:

COND = -2.

User subroutine including correlations "menu" which provide conductivity for several metals can be written in the following way:

SUBROUTINE COMETA(ICO,T,COND)

C**** ico=1 ==>inconel

C**** ico=2 ==>steel

C**** correlation selection for the considered metal

GO TO(10,20),ICO

C**** inconel conductivity

10 COND=9.872+.0153*T

RETURN

C****steel conductivity

20 COND=10.+.011*T

RETURN

END

To obtain that ICO assumes the right value, user must assign it to the COND variable preceded by the sign '-' in the file F14.

MODULE META

CARD F 

Residuals

The metallic wall whose model is included in META module, is subdivided in axial cells following the fluid flow coordinate.

For a number NCEL of cells, the first NCEL residuals associated to this module are referred to the energy balance in the metal. On the contrary, residuals from NCEL+1 to 2*NCEL are referred to heat conduction in the radial half-cell towards internal side of cylindrical wall. Last NCEL residuals are referred to heat conduction towards external side of the wall. All META residuals are normalized in temperature.
