MODULE WAND

CARD A

Model Generalities

CYLINDRICAL THICK METAL WALL

 

The WAND module simulates the thermal and extensometric operation of a steel body, having circular symmetry and with the lenght of simmetry axis prevailing if compared to radial dimensions.

It is radially submitted to convective thermal exchange, on internal and external walls; it is also possible to have imposed powers on these surfaces, and it is useful to introduce radiative transfer, i.e. calculated through STIR module.

Axially, WAND it is delimited by two bases, whose temperatures are set by corresponding input variables. At last, axial and radial conduction in the material is taken into account.

The steel body is divided into NCEL axial cells, each one consisting of NRAD radial layers. NCEL and NRAD are selected by the user during the block defining phase and then repeated in file F14.

NCEL is an integer between 2 and NMAX=10 while NRAD is an integer between 2 and RMAX=5.

WAND is a regulation module, that is the calculated variables are not included in the solving system of LEGO neither in steady nor in transient conditions, but once they are calculated they directly go out as results. This makes use of normalisation useless, so it is set at one for all the variables.

In transient condition, the bidimensional thermal field is determined through solution of differential equations with an implicit method assuming a time uncoupling among the temperature of axially adjacent cells. At each integration step, a tridiagonal matrix system results for each axial cell, and this system is solved irrespective of those relevant to the other axial cells through a replacement algorithm specific for tridiagonal systems. This system is obtained by eliminating the equations useful to the determination of temperatures of the two surfaces (written as algebraic links by the differential system), thus determining the state variables (temperature of steel layers). State temperatures calculated are then included in two (differential) equations relevant to two semilayers close to side walls (internal and external) in order to determine the surface temperatures.

In steady state, no uncoupling is made and all equations are solved at the same time according to a Gauss replacement method.

Interfacing with internal and external fluids can take place through coupling in flow rate, temperature and pressure, or flow rate, enthalpy and pressure. The selection is interactive and made inserting the model. In the first case enthalpy is obtained (if the calculation of convective exchange coefficient requires it) through the HEVAL iterative procedure. This last operates only in the range of overheated steam and in any case not in the two-phase zone, where it should be necessary to specify also the titre as additional parameter.

The type of correlation of convective exchange is selected on the basis of the value of parameters CODI (internal surface) and CODE (external surface) according to the following correspondence:

0

GAMMA is read by input (variables GMIi or GMEi)

1

A Dittus-Boelter correlation is used extended to the turbine rotor case

2

as case 1 for the steady state; in transient state a simplified correlation is used, which considers only dependence on the flow rate. All other dependences are frozen into a constant which is saved for each surface and for each cell in steady state.

Due to the already mentioned limitations connected to HEVAL, in the case the temperature is assigned as input along one or both the side surfaces and we are not in presence of overheated steam, it is necessary that CODI and/or CODE are equal to zero, that is corresponding to GAMMA must be read from input. Otherwise, if we want to use a correlation (Dittus-Boelter, Dittus-Boelter simplified, FORTRAN user auxiliary), it is necessary that the input is in enthalpy: an input in temperature would not have an adequate information content.

After calculation of the thermal field, the axial elongation of each cell consequent to the bidimensional thermal field and to the possible rotation (if rotor), is determined. Finally, the right DZDX and left DXDS elongation of the wall as a whole is highlights, referring to the grid point NPGF.

[K] TMij Temperature of the steel layer j of cell i
[K] TSIi Temperature of cell-i internal surface
[K] TSEi Temperature of cell-i internal surface
[m] DZSN Wall left elongation
[m] DZDX Right wall elongation
[K] TING Temperature axial input cell 1
[K] TOUT Temperature axial input last cell

[kg/s] WFIi flow rate fluid inside cell i
[Pa] PFIi pressure fluid inside cell i
[kg/s] WFEi flow rate of fluid outside cell i
[Pa] PFEi pressure fluid outside cell i

MODULE WAND

CARD C

Physical and Geometrical Data

 

In addition to the initial values of input and output variables of the WAND module, whose meaning has been defined in card B2, the user must assign the following set of data to complete the input required by the module:

Data relevant to the whole wall:

[ ] NCEL  Number of axial cells
[ ] NRAD Number of radial layers
[ ] CODI Type of correlation internal surface convective exchange
(values accepted: between 1 and 3)
[ ] CODE Type of correlation internal surface convective exchange
(values accepted: between 1 and 3)
[ ] ROTO Logical, true if rotor (values accepted 0, 1)
[m] DINT Thickness of fluid flowing channel along internal wall
(negative values not accepted; DINT = 0 means that the sliding area inside is not a circular crown but the whole circle)
[m] DEST Thickness of fluid flowing channel along external wall
(negative values not accepted
[W*m/K] COND steel thermal conductivity
[kg/m³] ROM steel density
[ ] ALFA Thermal expansion coefficient (default: 1.3E5)
[N/m²] ELAS Longitudinal elastic module (default: 2E11)
[ ] POIS Module of cross contraction (default: 0.3)
[ ] NPGF Number of grid point taken as reference for the calculation or right and left shifting (the first grid point is the left side of the left cell, while subsequent ones are separation lines between subsequent cells)

Data relevant to each single cell:

[m] LCEL cell-i length
[m] DIAI cell-i internal diameter (situations where RINTi=DIAIi/2-DINT<=0 are not accepted)
[m] DIAE cell-i external diameter
[kg] MMET cell-i steel mass (if MMETi <=0 the calculation of steel masses is made assuming cylindrical geometry)
[m²] SINT cell-i internal surface (if not assigned, it is calculated by the program assuming cylindrical geometry)
[m²] SEST cell-i external surface (if not assigned, is calculated by the program assuming cylindrical geometry)

Notes:

1) Interface radii among the layers of each single cell are calculated in order to have equal mass layers.

2) The specific heat of steels is calculated by linear interpolation of the value of iron specific heat in the range of temperatures 300K to 800K with the formula 0.484T+304.8 [K]

3) For DINT and DEST, the value to be assigned is not the distance in the radial direction of the steel wall, internal or external respectively, but the thickness of the fluid path free span. For instance, between a turbine stator and rotor, this thickness does not correspond to the actual rotor-stator distance but to the distance between rotor and stator blades forming the steam flowing channel. Such an error, could lead to overestimate the convective exchange coefficient, assigning unrealistic values of the Reynolds number.

MODULE WAND

CARD E 

User's Subprograms

 

To include in the module a correlation giving the thermal convective exchange coefficient between the channel internal wall and the fluid flowing inside or outside, it is possible to write a user program replacing the standard calculation when CODI or CODE are equal to 3.

The stating sentence is:

SUBROUTINE GAMPAM (W,P,H,TF,TM,OMEG,GAMMA)

 

Input parameters: W,P,H,TF,TM,OMEG

Output parameters: GAMMA

[kg/m³] W fluid flow rate
[Pa] P fluid pressure
[J/kg] H fluid enthalpy
[K] TF fluid temperature
[K] TM steel temperature
[rad/s] OMEG angular speed
[W/m²/K] GAMMA convective exchange coefficient (which shall be corrected with the value read by the input variable GMIi or GMEi according to the coefficient to be calculated)

MODULE WAND

CARD F 

Residuals

 

The residuals have the following expressions:

    K=1

    DO I=1,NCEL

    DO J=1,NRAD

        RNI(K)=TM(I,J)

        K=K+1

    ENDDO

    ENDDO

    DO I=1,NCEL

        RNI(K)=TSI(I)

        RNI(NCEL+K)=TSE(I)

        K=K+1

    ENDDO

    RNI(NCEL+K)=DZSN

    RNI(NCEL+K+1)=DZDX
