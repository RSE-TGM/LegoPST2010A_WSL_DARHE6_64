MODULE VENT

CARD A

Model Generalities

AXIAL & CENTRIFUGAL FAN

 

The VENT module simulates the functioning of a fan, both axial and centrifugal, complete of flow rate regulation system.

User has two different options to assign the fans characteristics (link among flow rate, head, speed and resisting power) for several positions of ALSR control parameter concerning the flow rate regulation system (see note 1):

1) to insert the fan manufacturer's characteristics (if they are known) by means of a user subprogram written according to specific methods (see below)

2) to choose among several characteristics included in the module, the one related to the concerned fan. In this case, since characteristics are not standardized, it is not enough that fan to simulate has geometrical features similar to that whose characteristics are chosen, but it needs that the fans are identical.

The module includes head curves concerning different values of control parameter (ALSR) related to the flow rate regulation system.

The power curves, instead, are approximated by means of a second degree polynomial whose coefficients are desumed from manufacturer's characteristics in correspondence of ALSR maximum value. In fact, manufacturer usually provides, for power, only such curve.

In addition, in determining the temperature gap between inlet and outlet of fluid flowing through the fan, the error made by using this characteristic (in substitution of the actual one) is negligible. It also needs to add that algebraical expression calculation, as regards to the chart data scansion and interpolation, produces a certain saving in calculation times.

Both head and power curves are obtained keeping the fan at nominal speed. The effective head and power for different speeds are calculated by means of the "fan equations" (see note 2).

For complex fan configurations, in the VENT module are included the following possibilities:

1) several identical fans in parallel configuration whose it is not interesting to study the unsymmetrical transients. User provides the physical and geometrical data concerning only one fan and specifies the number of them in parallel configuration

2) several identical fans in parallel configuration with the possibility to vary the number of them functioning under transient condition. It is not interesting to represent with accuracy the dynamics among fans in parallel configuration, but only to provide the correct flow rate or head variation to the fed air circuit. As in the preceding case, user varies (at the right moment) only the variable indicating the number of fans in parallel configuration

3) several identical (or not) fans in parallel configuration whose it is interesting to accurately represent the unsymmetrical transients. User has to utilize as many VENT module as fans are.

4) two or more fans in series configuration, in general, have to be dealt with interconnecting in series several VENT modules. It is enough an equivalent fan (only one VENT module), user must provide the consistent data: characteristic curves and nominal data.

In the adopted mathematical model, the following hypotheses are assumed:

- it is supposed that no fluid accumulation inside the fan occurs (inlet and outlet flow rates are considered equal each other)

- thermal capacity of the metal forming the fan and its possible thermal exchange with outside are not considered

- fan inertia is not considered (the speed is imposed from outside)

The used equations are:

1) pressure calculation equation at the fan outlet

2) thermal balance equation of fluid flowing through the fan.

The model keeps its validity even if a back-flow event occurs. Such phenomenon could happen, for example, in a suction fan because of a quick boiler depressurization caused by a sudden shutdown of the burners flame (boiler lock). Or, for example, in a pressurized boiler blowing fan as a consequence of its breakdown from feeding network.

Note 1

For axial fans the ALSR parameter is related to the position of blades, whereas for centrifugal ones the parameter is related to the position of the diffuser. In the module it is supposed that the ALSR value is zero when blades (or diffuser) position allows to flow through the fan the minimum flow rate. Value 1 is supposed when blades (or diffuser) position allows to flow through the fan the maximum flow rate. At intermediate values of ALSR control parameter, correspond intermediate values of blades (or diffuser) position.

Note 2

The so called "fan laws" state that the volumetric flow rate Wv, head DP and the absorbed power Qres are proportional respectively to the first power, to the square and to the cube of the OMVEn fan rotational speed.

Note 3

The perfect gas law is expressed by the following usual equation:

P*V = n*R*T

where:

- P is the gas pressure

- V is the gas volume

- T is the gas temperature

- R is the gas universal constant

- n is the mole number of the considered gas

Dividing by the gas mass M we get:

P/RO = (n*R/M)*T

where RO is the gas density.

Assuming:

where M/n is the gas molecular mass we obtain:

P/RO = ERRE*T

The above mentioned relation is used to determine the RO density of fluid flowing through the fan as a function of temperature and assigning to the pressure a constant value P = 101300. Pa.

MODULE VENT

CARD D

Module Variables Initialization

The NVEN variable value must be equal to the number of identical fans which are functioning at under rated conditions in parallel configuration. The variable initialization value must be consistent with such number. More precisely, the WFVE flow rate must be equal to the sum of the flow rates related to the single fans functioning simultaneously. On the contrary, physical and geometrical data must be those related to the single fan.

Under transient condition the NVEN input variable can be disturbed by user when simulates the variation of funs number functioning in parallel configuration. In simulating one ore more fan stopping, user must decrease (in steps) such input; on the contrary he must increase it if one or more fans starting has to be simulate.

N.B. NVEN must never cancel itself.

CARD E 

User's Subprograms

 

To assign the manufacturer's fan characteristics an user subprogram has to be provided. Its declarative statement is:

SUBROUTINE CARVENUT(TYP, OM, OM0, W, TET, RO, DELTAP, CR)

Input parameters: TYP, OM, OM0, W, TET, RO

Output parameters: DELTAP, CR

[ ]

TYP

corresponds to the TIPO data required in input (see above). It can be used as selection index of characteristics curves when only one user subprogram related to several fans type is requested.

[rad/s]

OM

fan angular speed

[rad/s]

OM0

nominal angular speed (reference speed provided by manufacturer)

[kg/s]

W

flow rate of fluid inside the fan

[p.u.]

TET

control parameter value of flow rate regulation system

[kg/m³]

RO

density of fluid flowing through the fan

[Pa]

DELTAP

fan head

[N*m]

CR

fan stall torque

As example of user subprogram, is represented the CARVENUT subroutine listing including the characteristics concerning the three type of fans mentioned above (in section on module's data):

SUBROUTINE CARVENUT(TYP, OM, OM0, W, TET, RO, DELTAP, CR)

C

C Significato di TIPO : 1 Prementi BRINDISI SUD

C 2 Ricircolo BRINDISI SUD

C 3 Aspirante BRINDISI SUD

C

ITY=-TYP

GO TO (10,20,30),ITY

10 CONTINUE

C

C TABELLE PER VENTILATORI PREMENTI (BRINDISI SUD)

C

CALL VE_PREM(OM, OM0, W, TET, RO, DELTAP, CR)

RETURN

20 CONTINUE

C

C TABELLE PER VENTILATORI DI RICIRCOLO (BRINDISI SUD)

C

CALL VE_RICI(OM, OM0, W, TET, RO, DELTAP, CR)

RETURN

30 CONTINUE

C

C TABELLE PER VENTILATORI ASPIRANTI (BRINDISI SUD)

C

CALL VE_ASPI(OM,OM0,W,TET,RO,DELTAP,CR)

RETURN

END

Residuals

The equations mentioned in first paragraph give rise to the following residuals:

RNI(1) = - PFVE + PFMV + DP

where DP is the fan head.

The residual is normalized as regards to the pressure PRIF = 1.E+05 Pa.

where:

- Cp is the specific heat of fluid flowing through the fan

- Qres is the fan resisting power.

The residual is normalized as regards to the temperature T0.

RNI(3) = - TFU1 + TFI1

The residual is normalized as regards to the temperature T0.

It is useful to note that last residual represents an identity rather than an equation.

In case of back-flow the expression of the residuals (2) and (3) are substituted by the following ones:

RNI(2) = - TFU2 + TFI2

The residual is normalized as regards to the temperature T0.

where:

- Cp is the specific heat of fluid flowing through the fan

- Qres is the fan resisting power.

The residual is normalized as regards to the temperature T0.

In this case it is useful to note that the residual (2) represents one identity rather than an equation.
