MODULE METP

CARD A

Model Generalities

CYLINDRIC METAL WALL

 

In the METP module the dynamics of the temperature in a thin metallic wall with plane geometry is represented, as a function of the exchanged thermal power at the boundary surfaces (internal and external).

The metallic wall whose behaviour has to be simulate, is subdivided in an arbitrary number of axial cells. Each of them is characterized by a given exchange surface and a certain volume of metal. Inside the cell heat transfer occurs by conduction in radial sense. The longitudinal conduction phenomenon is not considered.

As hypothesis it is assumed that the wall has a thin thickness in order to adopt a single radial cell representation.

For each axial cell the following equations are written:

1) the conservation equation of energy in the metal

2) the equation defining the dependence between the metal external wall temperature and the average temperature

3) the equation defining the dependence between the metal internal wall temperature and the average temperature.

The first equation is differential and defines the time variation of the metal average temperature as a function of the exchanged thermal powers.

The equations 2) and 3) are algebraical and represent (in steady-state conditions) the solution of the Fourier's equation for the considered geometry.

For each cell the module calculates the following variables:

1) metal average temperature

2) external wall temperature

3) internal wall temperature.

The METP module can be easily used (together with other ones included in the LEGO library) in building mathematical models related to plants components. For example, an evaporator model for a fossil fuel boiler can be built utilizing the following modules:

FUMB + METP + DRIS ==> flue gas duct cell + walls + fluid.

MODULE METP

CARD C

Physical and Geometrical Data

In addition to initial values of METP module input and output variables (meanings defined above), user must assign the following data to complete the file F14:

Data needed to define which sign is used to calculate the exchanged powers between the modules having in common the metallic wall

[ ] IPO1
if no data is written ==> power exchanged with internal side of the plane wall is calculated in accordance with the expression:
Qi = K*S*(TPi-TFi)
that is > 0. if internal fluid takes heat
else if IPO1 -1. ==> power exchanged with internal side of the plane wall is calculated in accordance with the expression:
Qi = K*S*(TFi-TPi)
that is > 0. if internal fluid gives up heat
[ ] IPO2
if no data is written ==> power exchanged with external side of the plane wall is calculated in accordance with the expression:
Qe = K*S*(TFe-TPe)
that is > 0. if internal fluid gives up heat
else if IPO2 -1. ==> power exchanged with external side of the plane wall is calculated in accordance with the expression:
Qe = K*S*(TPe-TFe)
that is > 0. if internal fluid takes heat

Needed physical data for the metal

[kg/m³] ROM metal density
[K/kg/C] CM metal specific heat
[W/m/C] COND metal conductivity

If metal conductivity depends on temperature, it is necessary to define the SUBROUTINE COMETP(ICO, TM, CON) (see the section regarding user subprograms).

Needed data to define the geometry of the plane wall

Each cell is defined through the following data:

[m³] VOLM metal wall volume
[m²] SUP1 metal wall internal surface
[m²] SUP2 metal wall external surface
[m] SPES metal wall thickness

When geometrical data concerning several cells are equal, it is enough to assign these data to the first cell leaving blank the fields related to the other cells.

We note that if the metallic wall is finned proper values of its surfaces and volume must be assigned.

MODULE METP

CARD E 

User's Subprograms

If metal conductivity depends on temperature, user must provide the following subroutine:

SUBROUTINE COMETP(ICO, T, COND)

Meaning of FORTRAN's symbols:

[ ] ICO pointer (integer value) in input to the subroutine which user can utilize to chose the function COND = F(T) among a "menu" of functions assembled in the subroutine itself
[K] T wall temperature provided in input to the subroutine
[W/m/C] COND conductivity calculated by the subroutine COMETP

E.g.: To assign the steel conductivity defined in the user subroutine COMETP, the following data has to be supplied in the file F14 for METP module which deals with the steel wall:

COND = -2.

The user subroutine including a correlations "menu" which provide conductivity for several metals can be written in the following way:

SUBROUTINE COMETP(ICO,T,COND)

C**** ico=1 ==>inconel

C**** ico=2 ==>steel

C**** correlation selection for the considered metal

GO TO(10,20),ICO

C**** inconel conductivity

10 COND=9.872+.0153*T

RETURN

C****steel conductivity

20 COND=10.+.011*T

RETURN

END

Since the subroutine COMETP(ICO, T, COND) is called only if COND < 0 in the file F14 and in this case the pointer ICO is set equal to - COND, to obtain that ICO assumes the right value (ICO = 2 in the example above), user must assign it preceded by the sign '-' to the COND variable in the file F14

MODULE METP

CARD F 

Residuals

 

The plane metallic wall whose model is included in METP module, is subdivided in axial cells following the fluid flow coordinate.

For a number NCEL of cells, the first NCEL residuals associated to this module are referred to the energy balance in the metal. On the contrary, residuals from NCEL+1 to 2*NCEL are referred to heat conduction in the half-cell towards the internal side of the plane wall. Last NCEL residuals are referred to heat conduction in the external half-thickness of the wall.

All METP residuals are normalized with respect to the temperature.
