MODULE DENI

CARD A

Model Generalities

VARIABLES DENORMALIZER

 

In the DENI module, differently from most of LEGOCAD library modules, no equation has been coded. The module simply transfers the input variables values to correspondent output variables, using for the latter ones a unitary normalization factor. In short, therefore, a denormalization of the input variables is performed.

This operation is indispensable when some blocks dealing with regulation schemes and built up through the CONTROL CAD RG1 activity have to be used in a process LEGOCAD model.

We note that the DENI module has not to be used if the process control is performed with a LEGOCAD model set up only by means of regulation schemes.

In this case in fact, the value transfers from the process output variables to the regulation input variables are carried out by the SCHEDULER on the basis of the topological information supplied by the user in the file S01.DAT.

During the topology construction activity, for each output variable (max.20), the module requires four alphanumeric characters that will be used as the first four characters of the FORTRAN symbol adopted for each output variable.

The first two of such alphanumeric characters must be assigned respecting the LEGOCAD convention for the physical quantity which the variable represents.

According to this convention the following associations are given:

P Pressure
DP Pressure Difference
W Flow rate
Q Power
H Specific enthalpy
T Temperature
AL Valve stem position
O Angular velocity in rad/s
R Angular velocity in revolutions/min
M Mass
C Concentration
% Percentages
L Level
Y Level
RE Reactivity

In the following paragraph the four characters assigned during the topology definition are indicated with "U__i", where "U_" represent the first two characters identifying the physical nature of the variable, while "i" (i = 1,..., n) is the progressive order number of the considered variables.

The same convention is adopted for the input variables, just replacing the letter "U" with the letter "I".

MODULE DENI

CARD C

Physical and Geometrical Data

In addition to initial values of DENI module input and output variables (meanings defined above), user must assign the following data set to complete the module input on the file F14:

[ ] NORM_I__n alphanumeric value identifying the normalization factor of the input variables according to the following rule:
blank = 1.
W0 = default value for flow rates
H0 = default value for enthalpies
T0 = default value for temperatures
P0 = default value for pressures
Q0 = default value for powers (= W0 * H0)
RO0 = default value for densities
AL0 = default value for lengths
V0 = default value for volumes
DP0 = default value for pressure differences
numeric value = value to be used as normalization factor
