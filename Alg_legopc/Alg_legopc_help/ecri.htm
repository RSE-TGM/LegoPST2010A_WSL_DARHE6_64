MODULE ECRI

CARD A

Model Generalities

HEATER LIQUID/STEAM-FLUE GAS

 

The ECRI module calculates the thermodynamic regime of a heat exchanger, as a preheater liquid-flue gas or a reheater steam-flue gas. It can be used to solving two sorts of problem.

First, if we have the constructor characteristics of the heater, we can determine any regime. The aim is always to calculate the exchanger power and the fluid state in output.

Secondly, the module is able to dimension exchangers of power plant from a certain nominal regime wished

This module is issued from both Pepse code and dynamics library of legocad. Pepse is a reference code in the domain of calculation of thermal balance. So we use the same mode of modelling (the NUT method). Then, the Legocad modules EVAP and FUMN are used to simulating the dynamics of exchangers liquid/steam-flue gas. The stationary module ECRI must be helpful to the equivalent dynamics model (EVAP-FUMN) which may have problems to converge to some regimes. In these cases, we calculate them with the stationary model and determine the appropriate data. So both dynamics and stationary modelling must consider the same geometry.

The mathematical model is based of the following hypothesis :

The heat capacity of both fluids is constant along the exchanger.

The exchange coefficient is constant along the exchanger.

The exchange between the both fluids is a counterflow, a parallelflow or a crossflow.

The thermal exchange with the outside is neglectable.

Equations used are:

1) power calculation from the difference between input and output of gas temperature.

2) power calculation from the difference between input and output of fluid enthalpy.

3) equation for calculation of output pressure of liquid/steam.

4) calculation of mean temperature of tubes.

5) power calculation with NUT method.

6) calculation of exchange coefficient.

7) calculation of exchange surface.

8) calculation of fin efficiency.

9) calculation of exergetic rating performance

MODULE ECRI

CARD C

Physical and Geometrical Data

The user must assign the following set of geometrical data :

First, the global geometrical data :

[ ] TIPO= We distinguish the three following families of exchangers :
1 - counterflow
2 - parallelflow
3 - crossflow

All calculations are right for those exchanger types. But in realty, we find most often the following geometry :

Every row is a crossflow exchanger but the whole exchanger is not exactly crossflow but is similar to a counterflow. We know from experience that the exchanger behaviour is like a counterflow for more than 3 rows and like a crossflow for less than 3 rows.

 

[m]

DELZ

Height difference between input and output.

Secondly we have to assign the tubes geometry :

[m]

DIAM_EST

External diameter.

[m]

DIAM_INT

Internal diameter.

[ ]

FA

Factor of tubes arrangement (between 0 and 1) which serves for the calculation of the external convection factor. It is usually near 1 (for more information, see module FUMN documentation).

MODULE ECRI

CARD D

Module Variables Initialization

The initial values must be near the solution. It is important that the fluids state be correct at the calculation beginning (liquid or vapor).

MODULE ECRI

CARD E 

User's Subprograms

The module calls the subroutine UC12, UR12 (like FUMN), GAM01 or GAMTERT (like EVAP).

A correlation for the calculation of the external convection factor in case of longitudinal geometry is missing. The user might write a subprogram for this calculation.

MODULE ECRI

CARD F 

Residuals

The equations forming the module mathematic model, indicated in cards A1 and A2, respectively, provide a set of residuals, which expression, both extended and formal would require a number of comments beyond the scope of this manual. We therefore prefer to indicate consistancies between residuals and equations, mentioning also the relevant normalization factors and make reference to the manual section, relevant to the module: "Model library of the LEGO code for the simulation of power generation plants - Theory" for additional details.

RNI(1) - normalization factor T0.
calculation of the output temperature of flue gas.
RNI(2) - 

normalization factor H0

calculation of the output enthalpy of liquid or steam.
RNI(3) - normalization factor P0
relevant to the output pressure calculation of liquid or steam.
RNI(4) - normalization factor T0
relevant to the calculation of output temperature calculation of liquid or steam.
RNI(5) - normalization factor T0
relevant to the mean wall temperature.
RNI(6) - normalization factor W0*H0
relevant to the calculation of exchanger power.
RNI(7) - normalization factor 10000*T0/(W0*H0)
relevant to the calculation of global exchange coefficient.
RNI(8) - normalization factor 10000.
relevant to the calculation of the total exchanger surface (fin+tubes).
RNI(9) - normalization factor 1.
relevant to the calculation of efficiency of fin surface.
RNI(10) - normalization factor 1.
relevant to the calculation of exergetic rating performance.
