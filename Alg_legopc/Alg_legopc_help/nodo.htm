MODULE NODO

CARD A

Model Generalities

ONE LINEAR ALGEBRAIC EQUATION

 

In opposition to the most LEGOCAD library modules, in the NODO module the representative equations of a plant component mathematical-physic model are not encoded. In the NODO module, a simple linear algebraical equation is presented, like:

Aout*Xout + A(1)*X(1) + A(2)*X(2) + ... + A(n)*X(n) = 0.

Once the coefficients Aout, A(1), ... A(n) and the variable values X(1) ... X(n) have been assigned, the module determines the Xout variable value.

The variables that appear in the previous equation (max. 20, which means up to 19 inputs) may represent unhomogeneous quantities.

For each variable in fact, user must assign the corresponding normalization factor (see below) which allows first to denormalize them and then to renormalize them as regards to the normalization factor (NORMALIZ) assigned to the output variable.

In order to allow the correct connection with other LEGO modules (avoiding the emission of program warnings) the first character of the input and output variables can be choosen by the user.

In the simplest case, the NODE module can be used to add up algebraically different homogeneous quantities like, for example, the several pipe fluid flow rates flowing into a header.

PHYSICAL MEANING OF MODULE INPUT-OUTPUT VARIABLES

The first character of the input and output variables can be choosen by the user. In this case the name of the input and output variables are the following (where the "a" is replaced with the character assigned by the user):

[ ] aSOM is the calculated variable
[ ] aI_1 ... aI_n are the known variables.

If the user doesn’t assign the initial character, the default names are used for input and output variables:

[ ] USOM is the calculated variable
[ ] IN_1 ... IN_n are the known variables

MODULE NODO

CARD C

Physical and Geometrical Data

In addition to the initial values of input and output variables, user must assign the following data:

[ ]

NORMALIZ

the alphanumeric value identifying the normalization factor of calculated variable (see note 1);

[ ]

NORMI_i

the alphanumeric value identifying the normalization factor to use for denormalization of the i-th input variable (see note 1);

[ ]

(output var.)

the coefficient of variable to be calculated (see note 2);

[ ]

(i-th input var.)

the coefficient of known variables (see note 2).

 

Note 1: The alphanumeric values which have to be used to assign the normalization factors are the following:

W0 = flow rates default value

H0 = enthalpies default value

T0 = temperatures default value

P0 = pressures default value

Q0 = powers default value (W0*H0)

RO0 = densities default value

AL0 = lengths default value

V0 = volumes default value

DP0 = delta pressures default value

num. value = value to use as normalization factor

blank = for NORMALIZ: value = 1. (default for valve stem positions)

for NORMI_1: the NORMALIZ value

for NORMI_n: the NORMI_n-1 value

 

Note 2: The data symbols of the coefficients Aout, A(1), ..., A(n) of the relation presented in the card A are set equal to the name of the related variable. Therefore they depends on the first character assigned by the user and they will change due to inputs connections, as they will inherit the name of the connected output.

Note 3: As the variation of the symbol (due to variable name inheritance) inhibits the copy of the value by means of the EDI14 program in case of topological modification of a previously exixting model, the user must reassign the coefficient related to the newly connected (or differently reconnected) inputs of any NODO modules.
